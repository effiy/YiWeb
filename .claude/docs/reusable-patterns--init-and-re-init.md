# Reusable Patterns: Init / Re-init

> **Source**: `/generate-document init` T3 re-init run (2026-05-03)
> **Tags**: `init`, `re-init`, `port-drift`, `changelog`, `consistency-check`
> **Confirmations**: Single run observation; T1/T2 re-init may not trigger same patterns.

## Pattern 1: Structured fact drift detection

When code config changes (e.g., port 8000 -> 8080 in `src/core/config.js`), doc
references drift silently. Detection method: grep the old value across `docs/`
and cross-reference against the code source of truth.

**Evidence**: 3 docs had stale port 8000 (devops.md, FAQ.md,
project-init/04_usage-document.md). Fix in commit f849b67.

**Applicable boundary**: Projects where docs are maintained manually or via
code-generation without automated consistency hooks. Works for any structured
fact (port, path, component name, API endpoint).

## Pattern 2: Keep a Changelog format baseline

This project enforces [Keep a Changelog](https://keepachangelog.com) for
`docs/changelog.md`. Sections by date (YYYY-MM-DD), categorized by type
(feat/fix/refactor/docs/remove).

**Evidence**: changelog.md rewritten from ad-hoc list to structured format
during re-init (commit f849b67). Rule enforced in project-basics.md.

**Applicable boundary**: Any project with the `changelog.md` in the project
basics file list. T3 re-init enforces format; T1/T2 updates should preserve
existing structure.

## Pattern 3: Re-init post-verification checklist

After a T3 re-init, verify:

1. `grep 8000 docs/` -- zero stale port references
2. `test -f` for every path referenced in docs -- no dead links
3. Component list: compare `index.js` imports vs `ls` directory listing
4. Cross-reference anchor validity: check `[text](path)` targets exist

**Evidence**: All 4 checks passed in T3 re-init (see project-init/06_process-summary.md).

## Pitfall: Code-document fact drift

Root cause: documents are generated by reading code snapshots, but subsequent
code changes (config edits, directory renames) are not propagated to docs. The
drink accumulates silently until a re-init surfaces it.

**Prevention**: After any structural code change (config, directory, API), run
`grep` across docs/ for the old value. Better: add a pre-commit hook or CI
step that fails when a structured fact (port, path) in docs/ contradicts the
code source of truth.

## Pitfall: Changelog format inconsistency

Root cause: no enforced format rule at initial document creation. Ad-hoc
entries accumulate, making automated parsing impossible.

**Prevention**: The init `project-basics.md` rule now mandates Keep a Changelog
format. During re-init, the format check is part of the verification step.
