# 生成文档技能

## 快速开始

```bash
/generate-document 生成一份"功能名"的需求文档
/generate-document 生成一份"功能名"的需求任务
/generate-document 生成一份"功能名"的设计文档
/generate-document 生成一份"功能名"的使用文档
/generate-document 生成一份通用文档，主题内容...
/generate-document 生成"简洁功能名-用户故事简短描述"的全文档
```

**全文档生成**：
- 触发命令: `/generate-document 生成"简洁功能名-用户故事简短描述"的全文档`
- 保存位置: `docs/00_rYr/简洁功能名/`
- 包含内容：
  1. 需求文档
  2. 需求任务
  3. 设计文档
  4. 使用文档
  5. 动态检查清单
  6. 项目报告

## 文档类型

| 类型 | 模板 | 规范 | 保存位置 | 依赖文档 |
|------|------|------|---------|---------|
| 需求文档 | ✅ | ✅ | `docs/01_需求文档/` | - |
| 需求任务 | ✅ | ✅ | `docs/02_需求任务/` | 需求文档 |
| 设计文档 | ✅ | ✅ | `docs/03_设计文档/` | 需求任务 |
| 使用文档 | ❌ | ✅ | `docs/04_使用文档/` | 设计文档 |
| 项目报告 | ❌ | ✅ | `docs/05_项目报告/` | 设计文档 |
| 通用文档 | ✅ | ✅ | `docs/` | - |

## 核心工作流

1. **解析需求** - 识别文档类型、功能名、参考文档
2. **查找模板** - 在 `templates/` 中查找对应模板
3. **加载规范** - 在 `rules/` 中加载文档规范
4. **加载检查清单** - 在 `checklists/` 中加载检查清单
5. **读取上游文档** - 如果有依赖关系，读取前一阶段文档
6. **生成内容** - 遵循模板和规范生成文档
7. **质量自检** - P0/P1/P2 检查
8. **可选：执行检查清单检查**
9. **可选：进入 Agents 审查**
10. **保存最终版本**

## 项目报告强制要求

项目报告必须包含：

1. **变更文件列表** - 所有修改文件的完整列表、路径、变更类型
2. **变更前后内容对比** - 每个变更文件的"变更前"和"变更后"内容对比
3. **变更汇总表** - 文件路径、变更类型、影响评估、关键变更说明

## 动态检查清单

生成全文档时会自动创建动态检查清单，基于：

1. 从需求任务提取的主要操作场景
2. 从设计文档提取的实现细节
3. 使用 `find-skills` 查找合适的验证技能
