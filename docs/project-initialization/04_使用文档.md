## 功能介绍

项目初始化功能用于为 YiWeb 项目建立完整的文档体系和开发规范，通过 `/generate-document init` 命令一键生成项目基础文件和全文档集模板。

### 核心特性

🎯 规范驱动生成 - 严格遵循 .claude/ 下的规则和清单
⚡ 代码 Grounding - 所有内容基于实际项目代码，不虚构
📖 完整文档集 - 生成项目基础文件和 docs/项目初始化/ 全文档集

### 适用人群

- 新加入项目的开发者
- 使用 Claude Code 的 AI 辅助开发者
- 需要建立文档规范的项目维护者

## 快速开始

### 前置条件

- [ ] YiWeb 项目仓库已克隆到本地
- [ ] Claude Code 环境已配置
- [ ] .claude/skills/generate-document/ 目录存在且包含规范文件

### 30 秒上手

1. 打开 Claude Code，进入 YiWeb 项目目录
2. 执行命令：`/generate-document init`
3. 等待生成完成
4. 检查生成的文件：`ls -la docs/`

🎉 完成！项目基础文档和全文档集已生成。

## 操作场景

### 📋 场景 1：为新项目初始化文档

**适用情况**：刚创建 YiWeb 项目，需要建立完整的文档体系。

**操作步骤**：
1. 确认项目根目录正确
2. 在 Claude Code 中执行：`/generate-document init`
3. 等待技能执行完成
4. 检查生成的文件：
   - 根目录下的 CLAUDE.md、README.md
   - docs/ 目录下的 6 个基础文档
   - docs/项目初始化/ 目录下的 7 个文档集

**预期结果**：所有文件按规范生成，内容基于项目实际代码 grounding。

**注意事项**：
✅ 确认项目目录结构正确
✅ 不确定的内容会标注"待补充（原因：…）"
❌ 不要手动修改生成的文档结构（内容可调整）

### 🎨 场景 2：更新现有项目的基础文档

**适用情况**：项目已有一段时间，需要更新基础文档以反映最新代码结构。

**操作步骤**：
1. 备份当前重要文档（如需要）
2. 执行：`/generate-document init`
3. 对比变更内容
4. 确认是否接受更新建议

**预期结果**：技能会检查现有文件一致性，不一致时提出更新建议。

**注意事项**：
✅ 执行前可先用 git 暂存当前修改
✅ review 变更内容后再提交
❌ 不要盲目覆盖已有文档

### 🔧 场景 3：参考全文档集开发新功能

**适用情况**：准备开发新功能，需要文档模板参考。

**操作步骤**：
1. 查看 docs/项目初始化/ 下的文档结构
2. 复制文档结构到新功能目录
3. 按实际需求修改内容
4. 或直接使用：`/generate-document <功能名>-描述`

**预期结果**：新功能有完整的文档结构参考。

**注意事项**：
✅ 新功能文档放在 docs/<功能名>/ 目录
✅ 使用 /generate-document 命令保证规范
❌ 不要随意修改文档章节结构

## 常见问题

### 💡 基础问题

#### Q: 为什么执行 /generate-document init 后有些内容标注"待补充"？

**症状**：生成的文档中很多章节写着"待补充（原因：…）"

**原因**：防幻觉机制，当无法从代码中推断确切内容时，不虚构，而是标注待补充。

**排查步骤**：
1. 检查项目代码是否完整
2. 确认标注的原因是否合理

**修复方案**：
- 可以手动补充待补充的内容
- 或继续开发，后续功能丰富后重新生成

**预防措施**：无，这是正常的防幻觉行为。

#### Q: 生成的文档可以修改吗？

**症状**：需要调整文档内容以符合项目实际情况。

**原因**：自动生成的内容可能需要人工补充和调整。

**修复方案**：
- 可以修改文档内容
- 但建议保持章节结构与规范一致
- 修改后可以 git 提交

### ⚙️ 高级问题

#### Q: 如何自定义文档生成规范？

**症状**：需要调整生成文档的格式和内容。

**原因**：项目有特殊的文档规范要求。

**修复方案**：
- 修改 .claude/skills/generate-document/rules/ 下的规范文件
- 修改 .claude/skills/generate-document/checklists/ 下的检查清单

#### Q: generate-document 技能还能生成其他功能的文档吗？

**症状**：需要为新功能生成文档。

**原因**：项目有新功能开发需求。

**修复方案**：
- 使用：`/generate-document <功能名>-描述`
- 参考 docs/项目初始化/ 的文档结构

### 🔧 故障排查

#### Q: 执行命令报错，找不到规范文件？

**症状**：执行 /generate-document init 时报错，提示找不到 rules/ 或 checklists/。

**原因**：.claude/ 目录或其中的技能文件缺失。

**排查步骤**：
1. 检查 .claude/skills/generate-document/ 目录是否存在
2. 检查 rules/ 和 checklists/ 目录是否存在

**修复方案**：
- 从 git 恢复 .claude/ 目录
- 或重新克隆项目

#### Q: 生成的文档中路径不正确？

**症状**：文档中引用的文件路径不存在。

**原因**：grounding 过程可能有误，或项目结构发生变化。

**排查步骤**：
1. 验证路径是否真实存在：`ls /path/to/file`
2. 检查项目结构是否有变化

**修复方案**：
- 手动修正路径
- 或重新执行 /generate-document init

## 技巧与提示

### 💡 实用技巧

1. **先备份再生成**：更新文档前，先用 git 暂存或提交当前状态
2. **逐步补充**：标注"待补充"的内容可以逐步完善，不必一次完成
3. **保持结构一致**：修改内容时尽量保持规范要求的章节结构
4. **参考示例**：docs/项目初始化/ 是很好的参考模板

### ⌨️ 快捷键（如适用）

（项目初始化无特定快捷键）

### 📚 最佳实践

1. **版本化文档**：将文档纳入 git 版本控制，记录变更历史
2. **及时更新**：项目架构变更后，及时更新相关文档
3. **规范先行**：开发新功能前，先用 /generate-document 生成文档
4. **验证链接**：定期检查文档中的链接是否有效
