## 功能介绍

本功能用于从 YiWeb 项目中彻底移除所有与新闻相关的功能、页面和组件，简化项目结构，减少不必要的维护成本。移除范围包括新闻页面视图、NewsList 业务组件，以及引用这些组件的入口文件，同时保留仍在 AICR 页面中使用的 SearchHeader 组件。

🎯 **安全移除**：只删除明确不再使用的代码
⚡ **简化结构**：减少代码库复杂度
🔧 **完整清理**：移除所有相关文件，不留残留

**适用人群**：
- 项目维护者
- 需要简化项目结构的开发者

## 快速开始

### 前置条件

- [ ] YiWeb 项目仓库已克隆到本地
- [ ] Git 仓库当前状态可提交（无未提交的重要变更）
- [ ] 已确认新闻功能不再使用

### 30 秒上手

1. 执行 `git status` 检查当前状态
2. 按照操作场景执行删除和更新
3. 验证 AICR 功能正常
4. 提交变更

🎉 完成！项目结构已简化。

## 操作场景

### 📋 场景：执行完整移除流程

**适用情况**：需要从项目中移除新闻功能，简化代码库。

**操作步骤**：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | `git status` | 检查当前 Git 状态 |
| 2 | `git add . && git commit -m "backup before remove news"` | 备份当前状态（可选但推荐） |
| 3 | `rm -rf src/views/news/` | 删除新闻视图目录 |
| 4 | `rm -rf cdn/components/business/NewsList/` | 删除 NewsList 组件 |
| 5 | 编辑 `cdn/components/index.js` | 移除 NewsList 导出，保留 SearchHeader |
| 6 | 编辑 `README.md` | 移除新闻应用相关描述 |
| 7 | 启动本地服务器 | `python -m http.server 8000` |
| 8 | 访问 AICR 页面 | `http://localhost:8000/src/views/aicr/index.html` |
| 9 | 验证功能正常 | 测试 AICR 核心功能 |
| 10 | 提交变更 | `git add . && git commit -m "移除新闻功能"` |

**预期结果**：
- 新闻视图目录和 NewsList 组件已删除
- 组件注册入口和 README 已更新
- AICR 页面可正常访问和使用

**注意事项**：
✅ 确认 SearchHeader 组件在 AICR 页面中正常工作
✅ 删除前建议先备份当前状态
❌ 不要删除 SearchHeader 组件（AICR 页面使用）

### 🎨 场景：验证 AICR 功能未受影响

**适用情况**：移除操作完成后，需要验证其他功能不受影响。

**操作步骤**：

| 步骤 | 操作 | 验证点 |
|------|------|--------|
| 1 | 访问 AICR 页面 | 页面可正常加载 |
| 2 | 检查 SearchHeader 组件 | 组件正常显示 |
| 3 | 加载会话列表 | 会话列表可正常显示 |
| 4 | 选择文件 | 文件树可正常工作 |
| 5 | 测试聊天功能 | AI 聊天功能正常 |

**预期结果**：所有 AICR 功能完整，无任何异常。

**注意事项**：
✅ 重点验证 SearchHeader 组件是否正常
✅ 检查控制台是否有错误
❌ 不要忽略任何警告或错误

### 🔧 场景：回滚操作（如需要）

**适用情况**：如果发现移除操作有问题，需要回滚到之前的状态。

**操作步骤**：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | `git status` | 检查当前状态 |
| 2 | `git stash` 或 `git reset --hard HEAD` | 暂存或重置变更 |
| 3 | `git status` | 确认已回滚 |

**预期结果**：项目状态已恢复到移除操作之前。

**注意事项**：
✅ 回滚前确保没有其他重要变更
✅ 使用 `git stash` 可以保留变更以便后续参考

## 常见问题

### 💡 基础问题

#### Q: 为什么要保留 SearchHeader 组件？

**症状**：SearchHeader 组件似乎也是新闻功能的一部分。

**原因**：SearchHeader 组件在 AICR 页面中也有使用（`src/views/aicr/index.js:37,59`），删除会影响 AICR 功能。

**排查步骤**：
1. 检查 `src/views/aicr/index.js` 文件
2. 确认 SearchHeader 组件在第 37 行和第 59 行被引用

**解决方案**：
- 保留 SearchHeader 组件
- 只删除 NewsList 组件和新闻视图目录

**预防措施**：在删除前检查所有引用。

#### Q: 删除后如何验证是否成功？

**症状**：需要确认删除操作是否完整。

**原因**：需要确保所有相关文件都已删除，且其他功能不受影响。

**排查步骤**：
1. 检查 `src/views/news/` 目录是否不存在
2. 检查 `cdn/components/business/NewsList/` 目录是否不存在
3. 检查 `cdn/components/index.js` 是否不再包含 NewsList
4. 检查 README.md 是否不再包含新闻应用描述
5. 测试 AICR 页面功能

**解决方案**：按照验证步骤逐一检查。

### ⚙️ 高级问题

#### Q: 如何确认没有其他文件引用了新闻功能？

**症状**：担心有遗漏的引用。

**原因**：可能有未搜索到的文件引用了新闻功能。

**排查步骤**：
1. 使用 grep 搜索：`grep -r "news" --include="*.js" --include="*.html" --include="*.md" .`
2. 检查搜索结果，排除不相关的内容
3. 确认没有其他重要引用

**解决方案**：使用搜索工具全面检查。

#### Q: 删除后还需要做什么？

**症状**：完成删除操作后，不确定是否还有后续工作。

**原因**：可能还有其他相关的清理工作。

**解决建议**：
1. 更新变更日志（如果有）
2. 清理其他文档中的引用（如果有）
3. 确认所有功能正常后再合并到主分支

### 🔧 故障排查

#### Q: 删除后 AICR 页面出现错误怎么办？

**症状**：AICR 页面无法正常加载或功能异常。

**原因**：可能误删了某些重要文件或 SearchHeader 组件。

**排查步骤**：
1. 检查浏览器控制台错误信息
2. 确认 SearchHeader 组件是否被误删
3. 确认 `cdn/components/index.js` 中的 SearchHeader 导出是否保留
4. 检查 `src/views/aicr/index.js` 是否被误修改

**解决方案**：
- 如果 SearchHeader 被误删，从 Git 恢复
- 如果文件被误修改，从 Git 恢复
- 使用 `git checkout HEAD -- <file>` 恢复特定文件

**预防措施**：删除前先备份，删除后立即测试。

#### Q: Git 显示很多删除但不确定是否正确？

**症状**：Git 状态显示很多删除操作，需要确认。

**原因**：删除了整个目录，Git 会显示所有被删除的文件。

**排查步骤**：
1. 使用 `git diff --name-status` 查看变更列表
2. 确认删除的文件都是新闻功能相关的
3. 确认没有误删其他文件

**解决方案**：仔细检查 Git 变更列表。

## 技巧与提示

### 💡 实用技巧

1. **先备份再删除**：删除前先用 Git 提交或暂存当前状态
2. **使用 Git 查看变更**：使用 `git status` 和 `git diff` 检查变更
3. **分步验证**：每做一个变更就测试一下，避免最后发现问题
4. **搜索所有引用**：使用 grep 全面搜索确保没有遗漏
5. **保留 SearchHeader**：记住 SearchHeader 组件在 AICR 页面中也有使用

### 📚 最佳实践

1. **小步提交**：每个变更单独提交，便于回滚
2. **先测试再合并**：确认所有功能正常后再合并到主分支
3. **文档更新**：代码变更后及时更新相关文档
4. **团队通知**：如果是团队项目，通知团队成员变更内容
