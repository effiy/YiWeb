# 知识图谱可视化系统使用说明

## 概述

知识图谱系统已升级至 v2.0，支持**交互式可视化**。每个故事目录下的 `knowledge-graph.json` 文件包含完整的节点-边图结构，可通过故事面板直接查看。

## 核心功能

### 1. 知识图谱可视化（新）

- **打开方式**：在故事面板中，点击任意故事的「知识图谱」按钮（紫色图标）
- **交互操作**：
  - **缩放**：鼠标滚轮
  - **平移**：拖拽空白区域
  - **节点拖拽**：按住节点拖动到新位置
  - **选中节点**：单击节点高亮其所有连线，底部显示详情面板
  - **双击聚焦**：双击节点查看详情
  - **悬浮提示**：悬停节点显示名称、类型、描述
- **工具栏按钮**：
  - 🔄 重置视图：恢复默认布局和缩放
  - 🔤 切换标签：显示/隐藏节点名称
  - ⇱ 力导向布局：开/关自动布局物理引擎

### 2. 知识图谱数据加载
- 自动加载所有故事目录下的 `knowledge-graph.json` 文件
- 加载 `story-deps.json` 维护的故事依赖关系
- 建立统一的故事和场景数据模型

### 3. 文件-故事映射
- 自动建立文件路径到故事/场景的映射索引
- 支持精确匹配和模糊匹配
- 提供查询接口，可快速查找某个文件属于哪些故事和场景

## 数据结构 v2.0

### KnowledgeGraph JSON 格式

```json
{
  "version": "2.0.0",
  "generatedAt": "2026-05-30T12:00:00Z",
  "projectRoot": "docs/故事任务面板/yiweb-arch/",
  "graph": {
    "nodes": [
      {
        "id": "unique-node-id",
        "label": "显示名称",
        "type": "view|service|framework|utility|component|config|test|doc|external|...",
        "group": "L1-Views|L2-Services|L3-Framework|Tests|...",
        "file": "path/to/file.js",
        "description": "节点描述",
        "keyFunctions": ["fn1", "fn2"],
        "riskLevel": "🔴|🟡|🟢"
      }
    ],
    "edges": [
      {
        "source": "node-id-1",
        "target": "node-id-2",
        "relation": "imports|contains|depends_on|tests|re_exports|creates|triggers|...",
        "label": "关系描述"
      }
    ]
  },
  "story": {
    "name": "...",
    "description": "...",
    "scenarios": [...]
  }
}
```

### 节点类型 (type)

| 类型 | 颜色 | 说明 |
|------|------|------|
| `view` | #3B82F6 蓝 | 视图入口和页面组件 |
| `service` | #F59E0B 黄 | 服务层 API/数据操作 |
| `framework` | #8B5CF6 紫 | 框架层基础设施 |
| `utility` | #6366F1 靛 | 通用工具函数 |
| `component` | #06B6D4 青 | UI 组件 |
| `config` | #F59E0B 黄 | 配置文件 |
| `test` | #EF4444 红 | 测试文件 |
| `doc` | #6B7280 灰 | 文档 |
| `external` | #9CA3AF 灰 | 外部依赖 |

### 关系类型 (relation)

| 关系 | 颜色 | 说明 |
|------|------|------|
| `imports` | #94A3B8 | ESM import 语句 |
| `contains` | #3B82F6 | 包含/拥有子模块 |
| `depends_on` | #F59E0B | 功能依赖 |
| `tests` | #EF4444 | 测试关系 |
| `re_exports` | #10B981 | 聚合导出 |
| `triggers` | #EF4444 | 事件触发 |
| `mutates` | #F97316 | 状态变更 |
| `calls` | #6366F1 | 函数调用 |
| `layer_depends` | #94A3B8 | 分层依赖 |

## 缓存机制

- 系统会缓存已加载的知识图谱数据，避免重复请求
- 可通过 `refreshKnowledgeGraphCache()` 方法手动刷新缓存
- 缓存会存储在内存中，页面刷新后会重新加载

## 常见问题

### Q: 如何查看知识图谱？
A: 在故事面板中，点击故事卡片或详情面板中的紫色「知识图谱」按钮。

### Q: 新添加的故事为什么没有知识图谱？
A: 需要确保故事目录下存在 v2.0 格式的 `knowledge-graph.json` 文件，包含 `graph.nodes[]` 和 `graph.edges[]`。

### Q: 图谱不显示或报错？
A: 检查浏览器控制台日志（`[知识图谱]` 前缀），确认 JSON 文件格式正确、路径可访问。

### Q: 如何添加新的知识图谱？
A: 在 `docs/故事任务面板/<新故事目录>/` 下创建 `knowledge-graph.json`，遵循上述 v2.0 格式，并在 `story-deps.json` 中注册该故事。
