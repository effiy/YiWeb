## 功能概述

本功能将 AICR 页面中的 `session-list-tags`（会话标签筛选器）从当前位于侧边栏（fileTree 组件内）的位置，移动到顶部 header-row 的下一行，使其更加醒目且易于访问。这个调整将显著提升用户的标签筛选体验，让标签筛选功能成为页面的核心交互元素之一。

🎯 **更醒目的位置**：标签筛选器从侧边栏移到页面顶部，更加醒目
⚡ **更快的访问**：用户无需滚动或展开侧边栏即可快速筛选标签
📖 **更好的布局**：优化页面空间利用，让标签筛选成为一级交互

## 验收标准

### P0 - 必须通过
- 标签筛选器成功移动到 header-row 下方的新位置
- 所有原有标签筛选功能保持正常工作
- 标签筛选器在侧边栏收起时仍然可见且可用
- 页面布局没有明显错乱

### P1 - 应该通过
- 标签筛选器的样式与整体设计风格保持一致
- 在不同屏幕尺寸下有合理的响应式表现
- 标签筛选器的动画和交互反馈保持流畅

### P2 - 可以有
- 优化标签筛选器的视觉设计，使其更符合顶部位置的重要性
- 添加更丰富的标签交互效果

## 功能详情

### 位置调整
**当前位置**：位于侧边栏 fileTree 组件内（`src/views/aicr/components/fileTree/index.html:9-151`），仅在侧边栏展开且有标签时可见。

**目标位置**：位于 header-row 下方（紧邻 `<aicr-header>` 组件之后），始终可见（只要有标签数据）。

### 涉及组件
- **FileTree**：当前包含标签筛选器，需要移除这部分
- **SearchHeader / AicrHeader**：需要考虑是否在此组件中扩展
- **AicrPage**：主页面布局组件，需要容纳新的标签筛选器区域
- **AicrSidebar**：侧边栏组件，内部不再包含标签筛选器

### 数据流保持
标签筛选相关的状态管理（`selectedTags`、`tagFilterReverse`、`tagFilterNoTags`、`tagFilterExpanded`、`tagFilterSearchKeyword` 等）需要从 AicrPage 传递给新位置的标签筛选器组件。
