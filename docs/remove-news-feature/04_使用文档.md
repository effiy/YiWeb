## 功能介绍

### 功能说明

本文档描述如何**安全地从 YiWeb 项目中移除新闻相关功能**，包括新闻页面、NewsList 组件、SearchHeader 中的新闻按钮，以及工具函数中的新闻相关代码。

### 目标受众

- **项目维护者** - 需要清理代码库的开发者
- **技术负责人** - 需要了解改动范围和影响的决策者
- **DevOps 工程师** - 需要执行部署和验证的工程师

### 核心特性

🎯 **渐进式移除** - 分阶段执行，每一步都可验证  
⚡ **安全回滚** - 使用 Git 版本控制，随时可回滚  
🔧 **最小影响** - 仅删除必要代码，不影响核心功能  

### 谁应该使用

- 确认新闻功能不再需要的项目团队
- 需要精简代码库、降低维护成本的团队
- 准备进行项目重构的团队

## 操作场景

### 📋 场景 1: 完整的新闻功能移除流程

**适用情况**: 你需要一次性完成整个新闻功能的移除

**操作步骤**:

1. **准备阶段**
   ```bash
   # 确保当前分支是最新的
   git checkout main
   git pull
   
   # 创建新分支
   git checkout -b feature/remove-news
   ```

2. **删除新闻页面**
   ```bash
   git rm -r src/views/news/
   git commit -m "feat: remove news page directory"
   ```

3. **删除 NewsList 组件**
   ```bash
   git rm -r cdn/components/business/NewsList/
   # 手动修改 cdn/components/index.js 移除 NewsList 导出
   git add cdn/components/index.js
   git commit -m "feat: remove NewsList component"
   ```

4. **修改 SearchHeader**
   ```bash
   # 手动修改 cdn/components/business/SearchHeader/index.js
   # 移除 showNewsButton、newsHref props
   # 移除 openNews 方法
   git add cdn/components/business/SearchHeader/index.js
   git commit -m "feat: remove news button from SearchHeader"
   ```

5. **清理工具函数 - exportUtils.js**
   ```bash
   # 手动修改 cdn/utils/io/exportUtils.js
   # 移除 formatNewsItem 函数
   # 移除新闻导出相关代码
   git add cdn/utils/io/exportUtils.js
   git commit -m "refactor: remove news export from exportUtils"
   ```

6. **清理工具函数 - domain.js**
   ```bash
   # 手动修改 cdn/utils/data/domain.js
   # 移除新闻特定的注释和别名
   git add cdn/utils/data/domain.js
   git commit -m "refactor: clean news-related code from domain.js"
   ```

7. **清理工具函数 - common.js**
   ```bash
   # 手动修改 cdn/utils/core/common.js
   # 移除 buildNewsSessionTags 函数
   # 从 keysToClean 中移除 'newsCache'
   git add cdn/utils/core/common.js
   git commit -m "refactor: remove news tag function from common.js"
   ```

8. **最终验证**
   ```bash
   # 运行构建
   # npm run build 或项目的构建命令
   
   # 检查 Git 状态
   git status
   git diff HEAD~7..HEAD  # 查看所有变更
   ```

9. **推送和 PR**
   ```bash
   git push -u origin feature/remove-news
   # 在 Git 平台创建 PR
   ```

**预期结果**:
- 所有新闻相关代码被移除
- 项目仍能正常构建和运行
- Git 历史清晰可追溯

**注意事项**:
✅ **正面提示**: 每一步提交后，使用 `git status` 检查状态  
❌ **负面警告**: 不要跳过构建验证，不要一次性提交所有改动

### 📋 场景 2: 分阶段安全移除

**适用情况**: 你希望更谨慎地执行，分阶段验证每一步

**操作步骤**:

1. **第一阶段：删除新闻页面和 NewsList 组件**
   ```bash
   git checkout -b phase1/remove-news-pages
   
   # 删除新闻页面
   git rm -r src/views/news/
   
   # 删除 NewsList 组件
   git rm -r cdn/components/business/NewsList/
   
   # 修改组件索引
   # 编辑 cdn/components/index.js
   
   # 提交
   git add -u
   git commit -m "phase1: remove news pages and NewsList component"
   
   # 验证构建
   # npm run build
   ```

2. **第二阶段：清理工具函数**
   ```bash
   git checkout -b phase2/clean-utils
   
   # 依次修改工具函数
   # 1. exportUtils.js
   # 2. domain.js
   # 3. common.js
   
   git add -u
   git commit -m "phase2: clean news-related code from utils"
   
   # 验证构建
   # npm run build
   ```

3. **第三阶段：修改 SearchHeader**
   ```bash
   git checkout -b phase3/update-search-header
   
   # 修改 SearchHeader
   # 编辑 cdn/components/business/SearchHeader/index.js
   
   git add -u
   git commit -m "phase3: remove news button from SearchHeader"
   
   # 验证构建
   # npm run build
   ```

4. **合并阶段分支**
   ```bash
   git checkout main
   git merge phase1/remove-news-pages
   git merge phase2/clean-utils
   git merge phase3/update-search-header
   ```

**预期结果**:
- 每一步都能单独验证
- 风险可控，可随时在任意阶段暂停
- 便于代码审查

**注意事项**:
✅ **正面提示**: 每个阶段完成后都进行完整测试  
❌ **负面警告**: 不要跨越阶段，确保前一阶段稳定后再进入下一阶段

---

### 📋 场景 3: 仅删除新闻页面（保留工具函数）

**适用情况**: 你只想删除新闻页面，但想保留工具函数中的通用功能

**操作步骤**:

1. **仅删除新闻页面和 NewsList 组件**
   ```bash
   git checkout -b remove-news-pages-only
   git rm -r src/views/news/
   git rm -r cdn/components/business/NewsList/
   
   # 修改 components/index.js 移除 NewsList 导出
   # 编辑 cdn/components/index.js
   
   git add -u
   git commit -m "feat: remove news pages only, keep utils"
   ```

2. **考虑是否修改 SearchHeader**
   - 选择 1: 也移除新闻按钮（推荐）
   - 选择 2: 保留新闻按钮但禁用或隐藏
   - 选择 3: 完全保留（不推荐，因为按钮指向的页面已删除）

3. **验证构建**

**预期结果**:
- 新闻页面和组件被删除
- 工具函数保持不变
- 构建正常

**注意事项**:
✅ **正面提示**: 如果选择保留 SearchHeader 中的新闻按钮，考虑添加重定向或隐藏  
❌ **负面警告**: 不要让用户点击新闻按钮后看到 404 错误

---

### 📋 场景 4: 回滚操作

**适用情况**: 移除后发现问题，需要回滚

**操作步骤**:

1. **如果还没有推送**
   ```bash
   # 查看提交历史
   git log --oneline
   
   # 回滚到移除前的状态
   git reset --hard <commit-before-removal>
   
   # 或者使用 revert（如果已经推送）
   # git revert <commit-hash>
   ```

2. **如果已经推送**
   ```bash
   # 创建 revert 提交
   git revert <commit-hash-1> <commit-hash-2> ...
   
   # 推送 revert
   git push
   
   # 创建 PR 合并 revert
   ```

3. **如果想保留分支但回到主分支**
   ```bash
   git checkout main
   # 分支还在，可以稍后回来
   ```

**预期结果**:
- 代码回到移除前的状态
- 构建正常

**注意事项**:
✅ **正面提示**: 使用 `git reflog` 查找丢失的提交  
❌ **负面警告**: 不要使用 `git push --force`，除非你完全确定

## 常见问题

### 💡 基础问题

**Q: 为什么要移除新闻功能？**

A: 新闻功能可能是历史功能，现在不再需要。移除它可以：
- 精简代码库
- 降低维护成本
- 减少构建时间
- 使项目更聚焦于核心功能

**Q: 如何确认新闻功能真的不再使用？**

A: 建议检查：
- 服务器访问日志 - 看新闻页面是否还有人访问
- 用户反馈 - 确认用户是否还在使用
- 业务需求 - 确认没有相关的业务依赖

**Q: 移除新闻功能会影响其他功能吗？**

A: 根据我们的分析，新闻功能相对独立。移除应该不会影响：
- SearchHeader 的其他功能
- 导出工具的每日清单和项目文件导出
- 其他组件和页面

但**请务必进行充分测试**以确认。

**Q: 这个改动需要数据库迁移吗？**

A: 不需要。这个改动仅涉及前端代码，不涉及数据库。

---

### ⚙️ 高级问题

**Q: 如何处理外部链接指向新闻页面的情况？**

A: 建议：
1. 检查服务器日志，确认是否还有外部链接
2. 如果有，考虑在服务器配置中添加重定向
3. 或者在删除前添加一个过渡期的公告页面

**Q: domain.js 中的通用功能真的可以保留吗？**

A: 是的。`domain.js` 中的函数大多是通用的域名处理功能，即使没有新闻页面，这些功能也可能对其他功能有用。我们建议保留它们，仅清理新闻特定的注释和别名。

**Q: 如何验证移除后一切正常？**

A: 建议的验证清单：
- [ ] 构建成功
- [ ] 无控制台错误
- [ ] SearchHeader 正常工作
- [ ] 每日清单导出正常
- [ ] 项目文件导出正常
- [ ] 其他核心功能正常
- [ ] 所有链接可正常跳转

**Q: 应该如何编写提交信息？**

A: 建议使用明确的提交信息，例如：
```
feat: remove news page directory
- 删除整个 /src/views/news/ 目录

feat: remove NewsList component
- 删除 NewsList 组件目录
- 更新组件索引文件

refactor: clean news-related code from utils
- 清理 exportUtils.js 中的新闻导出
- 清理 common.js 中的新闻标签功能
```

---

### 🔧 故障排查

**问题: 删除后构建失败，提示找不到模块**

A: 检查：
1. 是否还有其他文件在引用已删除的模块
2. 使用项目-wide 搜索检查
3. 检查 `components/index.js` 是否正确更新

**问题: SearchHeader 组件报错**

A: 检查：
1. 是否正确移除了 props 和方法
2. 是否有模板文件（template.html）也需要修改
3. 检查组件是否还在其他地方被使用

**问题: Git rm 命令失败**

A: 尝试：
1. 先确认文件存在：`ls -la <path>`
2. 检查文件是否已经被 Git 跟踪：`git status`
3. 可以直接删除后使用 `git add -u` 来暂存删除

**问题: 想回滚但找不到原来的提交**

A: 使用：
```bash
git reflog  # 查看最近的操作历史
git reset --hard HEAD@{n}  # 恢复到某个状态
```

## 技巧与提示

### 💡 实用技巧

1. **使用 Git 工作树测试**
   ```bash
   # 创建一个临时工作树来测试
   git worktree add ../test-remove-news main
   cd ../test-remove-news
   
   # 在这个工作树中测试移除
   # 完成后删除
   cd ..
   git worktree remove test-remove-news
   ```

2. **使用交互式变基整理提交**
   ```bash
   git rebase -i main
   # 可以合并、重排、重写提交信息
   ```

3. **使用 Git 对比工具查看变更**
   ```bash
   git difftool --dir-diff HEAD~7 HEAD
   # 使用可视化工具查看所有变更
   ```

4. **分阶段测试清单**
   ```bash
   # 创建一个简单的测试脚本
   # test-removal.sh
   echo "=== Step 1: Build ==="
   npm run build
   
   echo "=== Step 2: Check Git status ==="
   git status
   
   echo "=== Step 3: Check for dangling references ==="
   grep -r "news" --include="*.js" --include="*.vue" src/ cdn/ | head -20
   ```

5. **使用 `git add -p` 选择性暂存**
   ```bash
   # 如果你想在同一个文件中分多次提交
   git add -p cdn/utils/io/exportUtils.js
   ```

---

### 📚 最佳实践

1. **小步提交，频繁验证**
   - 每个逻辑单元一个提交
   - 每次提交后运行构建
   - 便于定位问题和回滚

2. **编写详细的 PR 描述**
   - 列出所有变更的文件
   - 说明影响范围
   - 列出验证步骤
   - 附上相关文档链接

3. **代码审查重点**
   - 确认没有遗漏的引用
   - 确认删除的文件确实不再需要
   - 确认保留的代码仍然正常工作
   - 检查是否有硬编码的新闻路径

4. **保留相关文档**
   - 即使代码删除了，保留本文档作为历史记录
   - 记录为什么要删除，以及何时删除的
