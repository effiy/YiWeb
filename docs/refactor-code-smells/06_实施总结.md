## 2. 阶段 1 详细记录：基础架构

### 2.1 常量提取 - constants.js

已创建 `cdn/utils/core/constants.js`，提取了以下常量：

- **超时时间配置** (TIMEOUTS): COMPONENT_LOAD, POLL_FAST, POLL_NORMAL, LOG_INTERVAL_SHORT, LOG_INTERVAL_LONG
- **防抖时间配置** (DEBOUNCE): SIDEBAR_RESIZE, SEARCH_INPUT
- **缓存配置** (CACHE): TEMPLATE_MAX_AGE
- **组件相关常量** (COMPONENTS): 侧边栏和聊天面板的 min/max/default 宽度
- **存储键名** (STORAGE_KEYS): AICR_SIDEBAR_WIDTH, AICR_CHAT_PANEL_WIDTH
- **事件名称** (EVENTS): 高亮相关事件
- **窗口暴露对象名** (WINDOW_EXPOSE_NAMES)
- **通用常量** (COMMON): 空字符串、分隔符等

**向后兼容**: 仍将所有常量挂载到 window 上。

### 2.2 注册中心 - registry.js

已创建 `cdn/utils/view/registry.js`，提供以下功能：

- **组件注册**: `register(name, component, options)` - 同时存储内部和挂载到 window
- **组件获取**: `get(name)` - 优先从内部获取，失败时回退到 window
- **组件存在检查**: `has(name)` - 双向检查
- **组件列表**: `list()` - 获取所有已注册组件名
- **组件卸载**: `unregister(name, keepWindow)` - 移除组件注册
- **等待组件加载**: `waitFor(name, eventName, timeout)` - 支持事件监听和轮询
- **批量等待**: `waitForAll(names, options)`
- **事件管理**: `dispatch()`, `on()`, `off()`, `once()`

**向后兼容**:
- 继续挂载组件到 window
- 同时通过 EventTarget 和 window 分发事件
- 从 window 回退读取组件

## 3. 阶段 2 详细记录：巨型文件拆分

### 3.1 拖拽逻辑提取 - resizer.js

已创建 `src/views/aicr/utils/resizer.js`，提取了：

- **createSidebarResizers(store)**: 创建 AICR 页面的两个侧边栏拖拽条
- **createResizer(sidebarElement, store, type, options)**: 创建单个拖拽条

**改进点**:
- 使用 constants.js 中的 COMPONENTS 常量替代硬编码数字
- 暴露了 cleanup() 函数用于清理事件监听器
- 保持完全相同的功能逻辑

### 3.2 事件监听器管理 - listenerManager.js

已创建 `src/views/aicr/utils/listenerManager.js`，提供：

- **createListenerManager()**: 创建监听器管理器实例
- **setupAicrEventListeners(store, options)**: 为 AICR 页面设置标准事件监听器

**管理器功能**:
- `add(eventName, handler, options)` - 添加监听器，返回 ID
- `remove(id)` - 按 ID 移除
- `removeByEvent(eventName)` - 按事件名移除
- `count()` - 获取当前数量
- `list()` - 获取列表
- `cleanup()` - 清理所有

**设置的监听器**:
- modalEscPressed
- keydown (ESC取消文件选择)
- highlightCodeLines

### 3.3 应用上下文容器 - context.js

已创建 `src/views/aicr/hooks/context.js`，提供：

- **createAppContext(store)**: 创建统一的上下文对象，封装所有常用依赖

**封装的内容**:
- 核心 store 和常用字段
- 工具函数 (safeExecute 等)
- 数据请求方法 (getData, postData 等)
- 服务配置 (SERVICE_MODULE, buildServiceUrl 等)
- 文件处理工具
- Markdown 渲染
- URL 构建
- 各种服务实例
- 认证工具
- 模型服务

**目的**: 简化 useMethods.js 中的长参数列表传递。

### 3.4 更新 aicr/index.js

已更新 `src/views/aicr/index.js`:

- 引入新模块 resizer.js 和 listenerManager.js
- 移除了内联的 createSidebarResizers 和 createResizer 函数（370-568 行）
- 移除了内联的事件监听器代码（303-367 行）
- 使用 setupAicrEventListeners(store) 替代
- 保持所有其他逻辑不变

**代码量减少**:
- 原文件: 657 行
- 新文件: 368 行
- 减少: 289 行 (44%)
