## 技术栈问题

### 模块加载失败

**症状**：控制台报错 `MODULE_LOAD_FAILED` 或 `Failed to fetch module`

**原因**：
- 模块路径错误
- 文件不存在
- 服务器未正确配置 MIME 类型
- CORS 问题

**排查步骤**：
1. 检查模块路径是否正确（使用绝对路径，以 `/` 开头）
2. 检查文件是否存在于仓库中：`ls /path/to/file.js`
3. 检查浏览器 Network 面板，查看请求是否成功
4. 检查服务器是否正确返回 `application/javascript` MIME 类型

**修复方案**：
- 修正模块路径
- 创建缺失的文件
- 配置服务器 MIME 类型（如 Nginx 的 `types` 配置）

**预防措施**：
- 使用真实存在的文件路径
- 新增模块前先确认目录结构

### 组件加载超时

**症状**：控制台报错 `COMPONENT_LOAD_TIMEOUT` 或组件长时间不显示

**原因**：
- 组件路径错误
- 模板文件不存在
- 组件定义错误
- 组件加载事件未触发

**排查步骤**：
1. 检查 `componentModules` 中的路径是否正确
2. 检查 `components` 列表中的组件名是否与定义一致
3. 检查组件是否正确触发了 `{ComponentName}Loaded` 事件
4. 检查浏览器控制台是否有组件相关错误

**修复方案**：
- 修正组件路径
- 使用 `registerGlobalComponent` 正确注册组件
- 确保组件触发了加载事件

### CORS 问题

**症状**：控制台报错 CORS 相关错误，API 请求失败

**原因**：
- 本地开发时跨域请求
- API 服务器未配置 CORS

**排查步骤**：
1. 检查 Network 面板的请求和响应头
2. 确认 API 服务器是否允许当前域名

**修复方案**：
- 使用 CORS 代理
- 使用浏览器插件禁用 CORS 检查（开发环境）
- 配置 API 服务器 CORS 策略

## 架构问题

### 视图工厂相关

**症状**：调用 `createBaseView` 时报错或应用不挂载

**原因**：
- 必需的 `createStore`、`useComputed`、`useMethods` 参数缺失
- 组件未正确加载
- DOM 元素 `#app` 不存在

**排查步骤**：
1. 检查 `createBaseView` 调用参数是否完整
2. 检查 HTML 中是否有 `id="app"` 的元素
3. 检查组件是否正确注册到 `window` 对象

**修复方案**：
- 补充必需参数
- 确保 HTML 中有正确的 DOM 元素
- 使用 `registerGlobalComponent` 正确注册组件

### 状态管理相关

**症状**：Store 中的数据不更新、响应式失效

**原因**：
- 未使用 `ref` 或 `reactive` 包裹数据
- 直接修改对象属性而未触发响应式更新
- 计算属性依赖未正确追踪

**排查步骤**：
1. 检查数据是否用 `ref` 或 `reactive` 包裹
2. 检查修改方式是否正确（`.value`）
3. 检查计算属性的依赖是否正确

**修复方案**：
- 正确使用 Vue 3 的响应式 API
- 通过 `.value` 修改 ref 数据
- 确保计算属性的所有依赖都在回调中访问

## AI 辅助开发问题

### 路径虚构

**症状**：代码中引用的文件或路径在仓库中不存在

**原因**：AI 虚构了不存在的路径或文件名

**排查步骤**：
1. 使用 `ls` 或 `Glob` 工具验证路径是否存在
2. 使用 `grep` 搜索相关文件是否有其他名称
3. 检查 git 历史是否有相关文件

**修复方案**：
- 使用真实存在的路径
- 或创建缺失的文件
- 或标注为待补充

**预防措施**：
- 编码前先扫描项目结构
- 使用 `Glob` 工具验证路径存在性
- 不确定时标注"待补充"而非虚构

### 重构约束

**症状**：AI 修改了与任务无关的代码或重构了没有问题的代码

**原因**：AI 过度优化或误解了任务范围

**排查步骤**：
1. 使用 git diff 检查变更范围
2. 确认哪些变更是任务必需的
3. 回滚不相关的变更

**修复方案**：
- 只修改任务相关的代码
- 不重构没有问题的代码
- 匹配现有代码风格

**预防措施**：
- 明确任务范围
- 修改前先检查 CLAUDE.md 的禁止事项
- 每步修改后验证是否在任务范围内
