# 01 需求文档 - 项目初始化

## 背景

YiWeb 是一个使用原生 JavaScript + Vue 3（CDN）构建的前端项目，专注于 AI 辅助代码审查。项目采用零构建架构，浏览器直接运行 ES Modules，所有外部依赖通过 CDN 加载。

## 目标

1. 建立完整的项目文档体系，覆盖架构、开发规范、运维指南
2. 定义清晰的技术栈和编码约定，确保团队协作一致性
3. 为后续功能开发提供标准化的文档模板和流程
4. 沉淀项目知识，降低新成员上手成本

## 范围

### 包含

- 项目基础文档（架构、状态管理、网络、DevOps、认证、安全、FAQ、变更日志）
- 初始化文档集（需求、设计、使用、清单、报告）
- 技术栈确认和目录结构定义

### 不包含

- 具体业务功能实现（如 AICR 的代码审查逻辑）
- 构建工具引入（保持零构建）
- 第三方依赖的源码修改

## 约束条件

- 无构建工具，直接使用浏览器原生 ES Modules
- Vue 3 通过 CDN 加载
- 禁止使用 npm/yarn 安装依赖
- 模块导入使用绝对路径（以 `/` 开头）

## 利益相关方

| 角色 | 职责 |
|------|------|
| 开发者 | 遵循规范进行开发，维护文档 |
| 维护者 | 审查文档更新，确保一致性 |
| 新成员 | 通过文档快速了解项目 |

## Postscript: Future Planning & Improvements

- 随项目演进定期刷新 init 文档
- 将常见问题持续补充到 FAQ
