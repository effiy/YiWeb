## §1 Story

**角色场景：** AICR 用户在浏览文件树时，需要在侧边栏内用二级目录标签筛选文件，同时期望侧边栏在页面滚动时保持可见（吸顶）。

**价值：**
- 侧边栏始终显示二级目录标签，用户可直接点击子目录名筛选文件
- 侧边栏吸顶确保文件树和标签筛选始终可操作，不随代码区滚动而消失

**范围边界：**
- 纯前端 UI/UX 改进，无后端接口变更
- 涉及 `fileTree/index.html`、`fileTree/index.css`、`fileTreeComputed.js`、`.aicr-sidebar` CSS 定位
- 复用现有 `AicrHeader` 标签组件的事件和状态（`selectedSessionTags`、`tagFilter*`）
- Sidebar 标签仅展示二级目录名，不展示一级目录名；一级目录筛选仍由 Header 负责
- 二级标签与一级标签联动：一级标签选中后，侧边栏显示对应一级目录下的二级标签；未选一级标签时隐藏二级标签区域；标签位置保持稳定，不因选中状态改变排序

**依赖：** 现有 `AicrHeader` 标签系统（拖拽排序、展开/折叠）已就绪

## §1.1 User Operations

**主操作流：**
1. 用户在 AICR 页面看到侧边栏（文件树视图）
2. 侧边栏顶部显示二级目录标签列表
3. 用户可直接点击二级标签筛选文件树
4. 一级目录标签仍通过 Header 选择，与 Sidebar 二级标签共用 `selectedTags`
5. 清除所有标签选中后，文件树恢复无筛选状态
6. 用户滚动代码区时，侧边栏保持吸顶可见

**UI 布局：**

```
┌──────────────────────────────────────────────────┐
│ AICR Header                                      │
│ [文件树] [代码]     [🏷 src] [cdn] [docs]  ← 一级标签 │
├────────────────┬─────────────────────────────────┤
│ Sidebar        │ Code Area                       │
│ (sticky 吸顶)  │                                  │
│ ┌────────────┐ │  // ... 代码内容 ...            │
│ │ 🏷 二级标签 │ │  ← 始终显示所有二级目录标签      │
│ │ [components]│ │    [components] [utils] [hooks]  │
│ │ [utils]    │ │                                  │
│ ├────────────┤ │                                  │
│ │ 📁 文件树  │ │                                  │
│ │ ├─ src/    │ │  ← 点击二级标签后文件树筛选      │
│ │ │ ├─ comp… │ │                                  │
│ │ │ ├─ utils │ │                                  │
│ │ │ └─ hooks │ │                                  │
│ └────────────┘ │                                  │
│ ◄─ 独立滚动 ─► │ ◄─ 页面滚动，sidebar 保持可见 ─► │
└─────────────────────────────────────────────────┘
```

> 窄屏（≤950px）：sidebar 变为顶部横条，不吸顶。

## §2 Requirements

| # | 功能点 | 输入 | 输出 | 错误行为 |
|---|-------|------|------|---------|
| F1 | 侧边栏始终显示二级目录标签 | `fileTree` 数据 | 渲染所有二级目录名作为标签 | 无二级目录时隐藏标签区域 |
| F2 | 标签点击筛选、展开/折叠 | 用户点击标签按钮 | emit `tag-select`（二级标签加入 selectedTags），`tag-filter-expand` | — |
| F3 | 侧边栏吸顶 | 页面滚动 | `.aicr-sidebar` 使用 `position: sticky; top: 0` | — |
| F4 | 二级标签筛选文件树 | 用户点击二级标签 | `sortedTree` 按二级标签过滤子目录 | 无匹配时显示空状态 |

**业务规则：**
- 侧边栏标签与一级标签联动：一级标签选中后显示对应一级目录下的二级标签，未选一级标签时隐藏二级标签区域
- 二级标签与一级标签共用 `selectedTags` 数组，`sortedTree` 基于标签名是否匹配一级目录名区分一级/二级分别筛选
- 点击二级标签不改变一级标签选中状态，二级标签仅用于过滤文件树中的二级目录
- 标签列表按文件数量降序排序，数量相同时按名称排序，不因选中状态改变标签位置
- 二级标签左对齐排列，字号缩小（10px）以区分层级
- Header 一级标签字号放大（16px，padding 10px 20px）以增强视觉层级和可点击区域
- 标签列表最大默认显示 8 个，超出显示展开按钮
- 侧边栏吸顶在 `.aicr-main` 容器内生效，sidebar 自身可独立滚动
- 侧边栏高度自适应铺满屏幕（`height: 100vh`），内部通过 flex 列布局分配空间
- 搜索框和二级标签区域吸顶固定在 sidebar 顶部（`.file-tree-sticky-header` 容器），不随文件树内容滚动

## §3 Design

### 技术设计

**模块 1 — fileTree 标签 UI（`fileTree/index.html` + `fileTreeLayout.css`）**

搜索框和二级标签列表包裹在 `.file-tree-sticky-header` 容器中，该容器 `position: sticky; top: 0` 吸顶在 sidebar 滚动区域内。标签列表包含：标签按钮列表（含「没有标签」按钮 + 展开/折叠），不显示搜索框。标签左对齐排列，字号 10px、padding 2px 6px 以缩小尺寸区分层级。

**模块 2 — fileTree 标签层级联动（`fileTreeComputed.js`）**

`allTags` 计算属性与一级标签联动：
- 识别选中的一级标签（匹配一级目录名）
- 未选一级标签时返回空数组（隐藏二级标签区域）
- 已选时遍历匹配的一级目录，收集其子目录名
- 去重、排序，应用用户自定义拖拽顺序

`tagCounts` 同步适配：仅统计选中一级目录下的二级目录文件数，未选一级标签时返回空。

`filteredTags` 稳定排序：标签按文件数量降序排列，数量相同按名称排序，不因选中状态改变位置。

`sortedTree` 区分一级/二级标签筛选：
- 一级标签筛选：基于标签名是否匹配一级目录名判断（匹配=一级标签，不匹配=二级标签）
- 一级标签筛选：过滤一级目录（Header 选择生效）
- 二级标签筛选：过滤子目录（只保留名称匹配的子文件夹，文件保留）
- 两级筛选独立应用，互不影响

**模块 3 — 侧边栏吸顶（`styles/index.css`）**

`.aicr-sidebar` 使用 `position: sticky; top: 0; align-self: flex-start; height: 100vh; display: flex; flex-direction: column;`。`height: 100vh` 确保 sidebar 始终铺满屏幕高度，内部通过 flex 列布局分配空间：`.file-tree-sticky-header`（吸顶）→ `.file-tree-content`（flex: 1，由 `.sidebar-scroll-container` 的 `overflow-y: auto` 提供独立滚动）。

`.aicr-main` 保持 `overflow: auto` 以确保 sticky 生效。

### 安全约束
- 无后端接口变更，无 XSS/SQL 注入风险
- 标签内容来自文件树数据（用户可控文件系统），需确保标签渲染使用 textContent 而非 innerHTML
