## §2 状态管理

复用现有 store 状态，不新增字段：

| 状态 | 来源 | 用途 |
|------|------|------|
| `selectedSessionTags` | Store | 一/二级标签共用选中数组 |
| `tagFilterSearchKeyword` | Store | 标签搜索关键词（Sidebar 中已移除搜索框，保留字段供 Header 复用） |
| `tagFilterExpanded` | Store | 标签展开/折叠 |
| `tagFilterReverse` | Store | 反向过滤（保留） |
| `tagFilterNoTags` | Store | 无标签筛选（保留） |

**层级判断逻辑：**
- 遍历 `selectedSessionTags`，检查每个标签是否存在于 `fileTree.value` 的一级目录名中
- 存在 → 一级标签；不存在 → 二级标签
- 当 `selectedSessionTags` 包含一级目录名时，`allTags` 返回该目录下的子目录名

## §3 关键实现决策

### D1 — allTags 仅返回二级目录

```js
allTags() {
    if (!Array.isArray(this.tree)) return [];

    // 始终收集所有二级目录名（跨全部一级目录）
    const tags = new Set();
    for (const item of this.tree) {
        if (item.type === 'folder' && Array.isArray(item.children)) {
            for (const child of item.children) {
                if (child.type === 'folder') tags.add(child.name);
            }
        }
    }

    const allTagsArray = Array.from(tags).sort();

    // 应用用户自定义拖拽顺序
    try {
        const saved = localStorage.getItem('aicr_file_tag_order');
        const savedOrder = saved ? JSON.parse(saved) : null;
        if (savedOrder && Array.isArray(savedOrder) && savedOrder.length > 0) {
            const orderedTags = savedOrder.filter(tag => tags.has(tag));
            const newTags = allTagsArray.filter(tag => !savedOrder.includes(tag));
            return [...orderedTags, ...newTags];
        }
    } catch (e) {
        console.warn('[FileTree] 加载标签顺序失败:', e);
    }

    return allTagsArray;
}
```

### D2 — sortedTree 二级筛选

```js
// 二级标签筛选逻辑（追加到现有 sortedTree）
const selectedTopLevel = ...; // 同上
const selectedSubTags = (this.selectedTags || []).filter(tag => !topLevelNames.has(tag));

if (selectedTopLevel.length === 1 && selectedSubTags.length > 0) {
    // 在一级筛选结果内，按二级标签过滤子目录
    for (const item of result) {
        if (item.type === 'folder' && item.children) {
            item.children = item.children.filter(child => {
                if (child.type === 'file') return true; // 文件保留
                if (child.type === 'folder') return selectedSubTags.includes(child.name);
                return true;
            });
        }
    }
}
```

### D3 — 吸顶实现

```css
.aicr-sidebar {
    position: sticky;
    top: 0;
    align-self: flex-start;
    height: 100vh;
    display: flex;
    flex-direction: column;
}
```

**约束：** `.aicr-main` 必须有 `overflow: auto`（或至少不设置 `overflow: hidden`），否则 sticky 不生效。

### D4 — 搜索+标签吸顶容器

搜索框和二级标签通过 `.file-tree-sticky-header` 包裹，在 sidebar 内部滚动时吸顶固定：

```css
.file-tree-sticky-header {
    position: sticky;
    top: 0;
    z-index: 10;
    background: var(--bg-secondary);
    flex-shrink: 0;
}
```

Sidebar 内部空间分配：`.file-tree-sticky-header`（吸顶，flex-shrink: 0）→ `.file-tree-content`（flex: 1，由 `.sidebar-scroll-container` 的 `overflow-y: auto` 提供独立滚动）。

### D5 — Header 一级标签放大

Header 一级标签与 Sidebar 二级标签通过差异化字号建立清晰视觉层级：

| 标签层级 | font-size | padding | 位置 |
|---------|-----------|---------|------|
| Header 一级标签 | 16px | 10px 20px | `sessionListTags/index.css` `.tag-item` |
| Sidebar 二级标签 | 10px | 2px 6px | `fileTreeTags.css` `.tag-item` |

一级标签字号从 12px → 16px，padding 从 6px 12px → 10px 20px，增强可点击区域和视觉主导地位。

## §4 样式隔离

- FileTree 标签样式使用 `.sidebar-tags-section` 前缀
- 复用 `.tag-item`、`.tag-expand-btn` 等已有类名（与 Header 标签样式一致）

- 吸顶样式在 `.aicr-sidebar` 上，无全局污染
