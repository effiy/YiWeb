## §1 使用场景

| 维度 | 内容 |
|------|------|
| **角色** | 正在开发新模块的功能开发者 |
| **前置** | 新模块需要引用其他模块 |
| **操作流** | 列出计划引用的全部模块 → 查依赖矩阵确认每项引用的方向合规 → 方向合规? → 确认不会形成循环依赖 → 会形成循环? → 提取公共依赖打破循环 / 依赖评估通过可添加 |
| **后置** | 新增依赖均合规，无反向依赖和循环依赖 |
| **异常** | 无法避免反向依赖 → 重新审视模块分层归属 |

## §2 技术评审

| 评审项 | 结论 | 说明 |
|--------|------|------|
| 方向约束完整性 | 通过 | L1→L2/L3 允许，反向禁止 |
| 循环检测方法 | 通过 | 拓扑排序检查 |
