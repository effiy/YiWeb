## §1 使用场景

| 维度 | 内容 |
|------|------|
| **角色** | 代码审查者 |
| **前置** | 有新增或修改的模块代码 |
| **操作流** | 收到代码变更 → 识别新增或修改的 import 语句 → 确定被 import 模块的类型 → 确定当前模块的类型 → 依赖方向符合规范? → 组件引用组件(允许) / 视图引用组件(允许) / 视图引用服务(允许) / 组件引用视图(违规) / 服务引用视图(违规) |
| **后置** | 所有跨模块引用通过方向校验 |
| **异常** | 发现组件引用视图 → 标记为架构违规，要求调整 |

## §2 技术评审

| 评审项 | 结论 | 说明 |
|--------|------|------|
| 方向约束明确 | 通过 | CDN 不依赖 src，通用组件不依赖业务逻辑 |
| 违规检测方法 | 通过 | grep CDN 模块中对 src/ 的 import |

### 依赖方向约束

| 方向 | 规则 | 违规示例 |
|------|------|---------|
| 通用组件 → 通用组件 | 允许 | YiIconButton → YiIcon |
| 通用组件 → 业务状态 | 禁止 | YiModal → aicr store |
| 基础设施 → 视图 | 禁止 | log.js → aicr/index.js |
| 渲染插件 → 通用组件 | 允许 | — |
| CDN → src | 禁止 | 任何 cdn/ 下文件 import src/ |
