## §1 使用场景

| 维度 | 内容 |
|------|------|
| **角色** | 需要修改公共模块的功能开发者 |
| **前置** | 已确定要修改的目标模块 |
| **操作流** | 确定要修改的模块 → 在模块地图中查找 → 模块在地图中? → 查看下游消费者列表 → 下游数量? → 0(直接修改) / 1-3(逐一验证) / >3(按影响级别分组) |
| **后置** | 明确变更影响范围，知道需要验证哪些下游 |
| **异常** | 模块在地图中但下游列表不完整 → 全局搜索 import 语句补充 |

## §2 技术评审

| 评审项 | 结论 | 说明 |
|--------|------|------|
| 模块三元组完整性 | 通过 | 每个模块记录入口+内部依赖+下游消费者 |
| 五大分类覆盖 | 通过 | 通用组件/渲染插件/基础设施/服务层/视图状态 |

### 五大分类与关键模块

| 分类 | 数量 | 关键模块 |
|------|:---:|------|
| 通用组件 | ≥14 | YiModal, YiButton, YiTag, YiLoading, YiIcon, HeaderActions, MarkdownView |
| 渲染插件 | ≥9 | SanitizePlugin, MermaidPlugin, TocPlugin, FrontmatterPlugin |
| 基础设施 | ≥11 | baseView, log, error, api, http, storage, eventBus, MarkdownRenderer |
| 服务层 | ≥7 | config, crud, requestHelper, authUtils, authErrorHandler |
| 视图状态 | 3 | aicr/hooks, claude/hooks, story/hooks |

### 模块三元组格式

| 元 | 含义 | 提取方法 |
|----|------|---------|
| 入口 | 模块主入口文件路径 | 读取目录下 index.js 或主文件 |
| 内部依赖 | 本模块 import 的其他模块 | 解析入口文件中的 import 语句 |
| 下游消费者 | import 本模块的其他模块 | 全局 grep 入口文件名 |
