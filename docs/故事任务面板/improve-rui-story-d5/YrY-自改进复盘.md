## §3 经验沉淀

### 发现

1. **D5 诊断敏感性**: 样本量小 (n=3) 时 33.3% 失败率可能被高估，需要更多数据确认
2. **工具路由可观测性不足**: 当前 execution-memory.jsonl 缺少工具调用粒度的成功/失败标记
3. **文档补齐价值**: 原提案仅故事任务文档，补齐 5 文档基线后形成了完整的改进→验证链条

### 建议

1. 在 execution-memory.jsonl 中增加 `tool_calls` 字段记录每次工具调用结果
2. D5 诊断增加最小样本量阈值 (n ≥ 10)，避免小样本误报
3. Agent 交接时传递工具调用上下文，便于下游 Agent 理解前置操作
