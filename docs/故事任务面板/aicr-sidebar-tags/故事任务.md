# 故事任务：aicr-sidebar-tags

## §1 Story

**角色场景：** AICR 用户在浏览文件树时，需要在侧边栏内直接用标签筛选文件（作为二级目录），同时期望侧边栏在页面滚动时保持可见（吸顶）。

**价值：**
- 侧边栏标签与 header-row 标签联动：header 选中一级目录后，侧边栏自动显示该目录下的子目录标签（下一层级）
- 侧边栏吸顶确保文件树和标签筛选始终可操作，不随代码区滚动而消失

**范围边界：**
- 涉及 `fileTree/index.html`（标签 UI + 去除 header-controls）、`fileTreeComputed.js`（allTags / tagCounts / sortedTree 层级感知）、`.aicr-sidebar` 的 CSS sticky 定位
- 标签数据流变更：`allTags` 根据 `selectedTags` 中的一级标签动态切换为二级子目录标签
- props/events 通路不变

**依赖：** `sessionListTags` 组件的 CSS 样式已在 `fileTreeTags.css` 中定义，可直接复用

## §1.1 User Operations

**主操作流：**
1. 用户在 AICR 页面看到侧边栏（文件树视图）
2. 侧边栏顶部依次显示：文件搜索框 → 标签筛选列表
3. 默认显示一级目录标签（与 header-row 一致）：用户点击标签进行文件筛选
4. 当用户在 header（或侧边栏）选中一级标签后，侧边栏标签自动切换为选中目录下的子目录（下一层级）
5. 用户可继续点击二级标签进一步筛选文件树
6. 清除所有标签选中后，侧边栏恢复显示一级目录标签
7. 用户滚动代码区时，侧边栏保持吸顶可见

**UI 布局：**

```
┌──────────────────────────────────────────────────┐
│ AICR Header                                      │
│ [文件树] [代码]     [🏷 src] [cdn] [docs]  ← 一级标签 │
├────────────────┬─────────────────────────────────┤
│ Sidebar        │ Code Area                       │
│ (sticky 吸顶)  │                                  │
│ ┌────────────┐ │  // ... 代码内容 ...            │
│ │ 🔍 搜索框  │ │                                  │
│ ├────────────┤ │                                  │
│ │ 🏷 二级标签 │ │  ← 选中 header「src」后显示子目录 │
│ │ [components]│ │    [src/components] [src/utils]  │
│ │ [utils]    │ │    [src/hooks]                  │
│ │ [hooks]    │ │                                  │
│ ├────────────┤ │                                  │
│ │ 📁 文件树  │ │                                  │
│ │ ├─ src/    │ │                                  │
│ │ │ ├─ comp… │ │  ← 可进一步点击二级标签筛选      │
│ │ │ ├─ utils │ │                                  │
│ │ │ └─ hooks │ │                                  │
│ └────────────┘ │                                  │
│ ◄─ 独立滚动 ─► │ ◄─ 页面滚动，sidebar 保持可见 ─► │
└─────────────────────────────────────────────────┘
```

> 窄屏（≤950px）：sidebar 变为顶部横条，不吸顶。

## §2 Requirements

| # | 功能点 | 输入 | 输出 | 错误行为 |
|---|-------|------|------|---------|
| F1 | 侧边栏标签与 header 标签联动，显示下一层级 | `allTags`（一级目录名），`selectedTags`（header 选中状态） | 无选中时渲染一级目录标签；有选中时渲染选中目录下的子目录标签 | 无标签时隐藏标签区域 |
| F2 | 标签点击筛选、搜索、展开/折叠 | 用户点击标签按钮 | emit `tag-select`（一/二级标签均加入 selectedTags），`tag-filter-search`，`tag-filter-expand` | — |
| F3 | 侧边栏吸顶 | 页面滚动 | `.aicr-sidebar` 使用 `position: sticky; top: 0` | — |

**业务规则：**
- 侧边栏标签层级联动：`selectedTags` 含一级目录名时，`allTags` 自动切换为该目录下的子目录名（二级标签）；清除后恢复一级
- 二级标签与一级标签共用 `selectedTags` 数组，`sortedTree` 区分一级/二级分别筛选
- 去除 header-controls 按钮栏（反向过滤、无标签筛选、清除按钮），保留标签搜索和展开/折叠
- 标签列表最大默认显示 8 个，超出显示展开按钮
- 侧边栏吸顶在 `.aicr-main` 容器内生效

## §3 Design

### 技术设计

**模块 1 — fileTree 标签 UI（`fileTree/index.html`）**

在 `file-tree-search-wrapper` 下方添加 `session-list-tags` 区域，复用 `fileTreeTags.css` 中已定义的样式。标签列表包含：搜索框 + 标签按钮列表（含「没有标签」按钮 + 展开/折叠）。

去除 `header-controls` 按钮栏（反向过滤、无标签筛选、展开、清除按钮不再在侧边栏显示）。

**模块 2 — fileTree 标签层级联动（`fileTreeComputed.js`）**

`allTags` 计算属性改为层级感知：
- `selectedTags` 中无一級目录名时 → 返回一级目录名（与 header 一致）
- `selectedTags` 中含一级目录名时 → 返回选中目录下的子目录名（二级标签）

`tagCounts` 同步适配：二级标签时统计子目录内文件数。

`sortedTree` 区分一级/二级标签筛选：
- 一级标签筛选：过滤一级目录（与原有逻辑一致）
- 二级标签筛选：在一级筛选结果内，过滤子目录（只保留名称匹配的子文件夹，文件保留）

**模块 3 — 侧边栏吸顶（`styles/index.css`）**

`.aicr-sidebar` 添加 `position: sticky; top: 0; align-self: flex-start; max-height: 100vh;`。

### 安全约束

- 标签列表内容来自 `allTags` prop（服务端数据），无用户直接 HTML 注入风险
- 使用 `v-for` 渲染，无 `v-html` 或 `innerHTML` 操作
- 不涉及新增 API 调用

### 影响链

| 变更文件 | 影响范围 | 二级传递 |
|---------|---------|---------|
| `fileTree/index.html` | 标签 UI 块 + 去除 header-controls | 无（纯展示，复用已有 props/events） |
| `fileTreeComputed.js` | `allTags` / `tagCounts` / `sortedTree` 层级感知 | `sortedTree` 二级筛选影响文件树展示 |
| `styles/index.css` | `.aicr-sidebar` sticky | 需验证响应式断点下行为正常（≤950px 时 sidebar 变为顶部横条） |
