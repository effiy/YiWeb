# 故事任务：aicr-sidebar-tags

## §1 Story

**角色场景：** AICR 用户在浏览文件树时，需要在侧边栏内直接用标签筛选文件（作为二级目录），同时期望侧边栏在页面滚动时保持可见（吸顶）。

**价值：**
- 标签筛选作为二级目录置于文件树搜索框下方，减少用户在 header 和 sidebar 之间的视觉跳转
- 侧边栏吸顶确保文件树和标签筛选始终可操作，不随代码区滚动而消失

**范围边界：**
- 仅涉及 `fileTree/index.html` 的标签 UI 新增和 `.aicr-sidebar` 的 CSS sticky 定位
- 不改变标签数据流（props/events 通路不变）
- 不改变标签业务逻辑

**依赖：** `sessionListTags` 组件的 CSS 样式已在 `fileTreeTags.css` 中定义，可直接复用

## §1.1 User Operations

**主操作流：**
1. 用户在 AICR 页面看到侧边栏（文件树视图）
2. 侧边栏顶部依次显示：文件搜索框 → 标签筛选列表（新增）
3. 用户点击标签项进行文件筛选，标签高亮选中态
4. 用户滚动代码区时，侧边栏保持吸顶可见
5. 用户仍可在 header 中使用相同的标签筛选（双入口同步）

**UI 布局：**

```
┌──────────────────────────────────────────────────┐
│ AICR Header                                      │
│ [文件树] [代码]                    [标签筛选入口]  │
├────────────────┬─────────────────────────────────┤
│ Sidebar        │ Code Area                       │
│ (sticky 吸顶)  │                                  │
│ ┌────────────┐ │  // ... 代码内容 ...            │
│ │ 🔍 搜索框  │ │                                  │
│ ├────────────┤ │                                  │
│ │ 🏷 标签筛选 │ │  ← 新增：标签列表（≤8 可见，    │
│ │ [all][P0]  │ │    超出展开），复用 fileTreeTags │
│ │ [P1][P2]   │ │    CSS                         │
│ ├────────────┤ │                                  │
│ │ 📁 文件树  │ │                                  │
│ │ ├─ src/    │ │                                  │
│ │ ├─ cdn/    │ │                                  │
│ │ └─ ...     │ │                                  │
│ └────────────┘ │                                  │
│ ◄─ 独立滚动 ─► │ ◄─ 页面滚动，sidebar 保持可见 ─► │
└────────────────┴─────────────────────────────────┘
```

> 窄屏（≤950px）：sidebar 变为顶部横条，不吸顶。

## §2 Requirements

| # | 功能点 | 输入 | 输出 | 错误行为 |
|---|-------|------|------|---------|
| F1 | 文件树搜索框下方显示标签列表 | `allTags`, `tagCounts`, `selectedTags` props | 渲染标签按钮列表，支持点击选中/取消 | 无标签时隐藏标签区域 |
| F2 | 标签搜索、反向过滤、无标签筛选、展开/折叠 | 用户点击按钮 | emit `tag-select`, `tag-clear`, `tag-filter-reverse`, `tag-filter-no-tags`, `tag-filter-expand`, `tag-filter-search` | — |
| F3 | 侧边栏吸顶 | 页面滚动 | `.aicr-sidebar` 使用 `position: sticky; top: 0` | — |

**业务规则：**
- 标签列表使用与 header 中相同的 `sessionListTags` UI 模式
- 标签列表最大默认显示 8 个，超出显示展开按钮
- 侧边栏吸顶在 `.aicr-main` 容器内生效

## §3 Design

### 技术设计

**模块 1 — fileTree 标签 UI（`fileTree/index.html`）**

在 `file-tree-search-wrapper` 下方添加 `session-list-tags` 区域，复用 `fileTreeTags.css` 中已定义的样式（`.session-list-tags`, `.tags-header`, `.tags-list`, `.tag-item` 等）。

数据绑定：fileTree 组件已接收所有标签相关 props（`selectedTags`, `tagFilterReverse`, `tagFilterNoTags`, `tagFilterExpanded`, `tagFilterSearchKeyword`），并已有 tag 相关 methods（`toggleTag`, `toggleReverse`, `toggleNoTags`, `toggleExpand`, `updateTagSearch`, `clearAllFilters`），直接绑定即可。

**模块 2 — 侧边栏吸顶（`styles/index.css`）**

`.aicr-sidebar` 添加 `position: sticky; top: 0; align-self: flex-start; max-height: 100vh;`。

`.aicr-sidebar` 的子元素 `.sidebar-scroll-container` 已有 `overflow-y: auto`，因此标签 + 搜索框会自然保持在侧边栏顶部可见，文件树内容在下方滚动。

### 安全约束

- 标签列表内容来自 `allTags` prop（服务端数据），无用户直接 HTML 注入风险
- 使用 `v-for` 渲染，无 `v-html` 或 `innerHTML` 操作
- 不涉及新增 API 调用

### 影响链

| 变更文件 | 影响范围 | 二级传递 |
|---------|---------|---------|
| `fileTree/index.html` | 新增标签 UI 块 | 无（纯展示，复用已有 props/events） |
| `styles/index.css` | `.aicr-sidebar` sticky | 需验证响应式断点下行为正常（≤950px 时 sidebar 变为顶部横条） |
