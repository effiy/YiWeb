# 前端技术评审：aicr-sidebar-tags

## 评审结论

前端改动涉及 2 个文件：
1. `fileTree/index.html` — 在搜索框下方新增标签筛选列表 UI
2. `styles/index.css` — `.aicr-sidebar` 添加 sticky 定位

不新增组件、不改变数据流、不复用 `sessionListTags` 组件（改为内联模板以保持 fileTree 独立）。

## 组件树

```
aicrPage
├── aicrHeader (已有标签列表 — 保持不变)
└── .aicr-main
    └── aicrSidebar
        └── fileTree (新增标签 UI)
            ├── .file-tree-search-wrapper (已有)
            ├── .session-list-tags (新增) ← 标签筛选
            └── .file-tree-content (已有)
```

## Hooks 状态管理

不新增 hooks。标签状态通过已有 props 传入 fileTree：
- `allTags` — 全部标签（来自 store）
- `selectedTags` — 已选标签
- `tagFilterReverse` — 反向过滤开关
- `tagFilterNoTags` — 无标签筛选开关
- `tagFilterExpanded` — 展开状态
- `tagFilterSearchKeyword` — 标签搜索关键词

事件 emit 通路不变：`tag-select → aicrSidebar → aicrPage → store`

## 交互细节

| 交互 | 行为 |
|------|------|
| 点击标签 | 选中/取消，emit `tag-select` |
| 反向过滤按钮 | 切换反向模式 |
| 无标签按钮 | 筛选无标签文件 |
| 展开按钮 | 展开/折叠全部标签（默认显示 8 个） |
| 标签搜索 | 输入关键词过滤标签列表 |
| 清除过滤 | 清除所有标签筛选条件 |

## 样式隔离

- 标签 UI 复用 `fileTreeTags.css` 中已有的 `.session-list-tags` 系列样式
- 不引入新的 CSS 文件
- `.aicr-sidebar` 新增 `position: sticky` 仍使用项目已有的 `.aicr-sidebar` 类名
- 窄屏 ≤950px 时 sticky 被原有 `position: relative` 覆盖（保持现有行为）

## 加载顺序

fileTree 组件 CSS 通过 `index.css` 的 `@import` 链加载（已有 `fileTreeTags.css`），无需变更 manifest。
