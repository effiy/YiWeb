## 2. 架构指标

### 2.1 六维推演

| 维度 | 评估 | 说明 |
|------|------|------|
| 模块耦合 | 低 | clearCacheMethods 为独立工具函数，无组件耦合 |
| 接口稳定性 | 高 | `window.confirm` + `localStorage` 为浏览器标准 API，稳定 |
| 扩展性 | 好 | 保留键白名单为 Set，增删键名只需修改常量 |
| 可测试性 | 好 | 纯函数，无外部依赖，可单元测试 |
| 安全边界 | 清晰 | 仅操作当前域名 localStorage，保留键白名单硬编码 |
| 依赖方向 | 单向 | clearCacheMethods → 浏览器 API，无反向依赖 |

## 4. 经验沉淀

| # | 经验 | 类别 | 适用范围 |
|---|------|------|---------|
| 1 | 通用组件扩展通过 prop 控制显示，默认关闭，不影响其他消费者 | 技术决策 | 所有 Vue 组件库扩展 |
| 2 | localStorage 遍历清理时，先收集全部键再移除，避免索引漂移 | 技术决策 | 批量 localStorage 操作 |
| 3 | 跨项目引用需求文档时，需验证存储键名、路径、架构是否匹配目标项目 | 流程优化 | 所有跨项目文档复用 |
| 4 | 浏览器 CacheStorage API 为异步 Promise，涉及缓存的清理操作必须 await 所有删除完成后再执行页面导航，否则产生竞态 | 技术决策 | 所有缓存清理+页面跳转场景 |
| 5 | `location.reload(true)` 的 forceGet 参数已被现代浏览器忽略，清除 HTTP 缓存应在 CacheStorage 清除后直接 `location.reload()` 重新加载页面 | 技术决策 | 所有需要绕过浏览器 HTTP 缓存的硬刷新场景 |
| 6 | 按钮图标应与功能语义一致：刷新使用 `fa-rotate-right`，删除使用 `fa-trash-can`，不同功能不同图标，避免用户认知混淆 | 技术决策 | 所有 UI 按钮图标的语义选择 |
