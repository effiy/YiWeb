## 1. 存储架构

### 1.1 localStorage 键值设计

| 键名 | 类型 | 用途 | 保留策略 |
|------|------|------|---------|
| `YiWeb.apiToken.v1` | string | API 认证 Token | **必须保留** |
| `YiWeb.apiModel.v1` | string | Ollama 模型选择 | **必须保留** |
| `env` | string | 环境配置（local/prod） | 清除 |
| `aicrSidebarWidth` | string | AICR 侧边栏宽度 | 清除 |
| `aicrChatPanelWidth` | string | AICR 聊天面板宽度 | 清除 |
| `debug` | string | 调试标志 | 清除 |
| `{业务键}` | 任意 | 应用运行时缓存 | 清除 |

> 保留键列表集中定义在 `cdn/utils/core/constants.js` 的 `STORAGE_KEYS` 中。清除逻辑遍历全部 localStorage 键，匹配保留列表的跳过，其余移除。

### 1.2 存储操作契约

```
clearCacheAndRefresh()
  ├─ 遍历 localStorage (i=0..length-1)
  ├─ 获取 key = localStorage.key(i)
  ├─ 若 key 在 PRESERVE_KEYS 中 → 跳过
  ├─ 否则 → localStorage.removeItem(key)（异常静默忽略）
  └─ 完成后 → location.reload()
```

| 操作 | 原子性 | 失败处理 |
|------|--------|---------|
| removeItem | 单键原子 | try/catch 静默忽略，继续下一键 |
| reload | 页面级 | 在全部 removeItem 完成后执行 |

## 4. 安全约束

| # | 威胁 | 信任边界 | 缓解措施 | 优先级 |
|---|------|---------|---------|--------|
| 1 | 误删 Token 导致用户需重新鉴权 | 用户操作边界 | 保留键白名单硬编码，遍历移除前强制过滤 | P0 |
| 2 | 恶意脚本通过 XSS 触发缓存清除 | 脚本注入边界 | 功能绑定用户显式点击 + confirm 确认，无自动触发路径 | P0 |
| 3 | localStorage 遍历期间其他脚本并发修改 | 浏览器单线程 | JavaScript 单线程执行模型天然避免并发竞争 | P1 |
| 4 | 保留键名变更导致误清除 | 代码变更边界 | 保留键常量集中定义在 `cdn/utils/core/constants.js`，变更需代码审查 | P1 |

## 5. 性能与限制

| 维度 | 约束 | 应对 |
|------|------|------|
| localStorage 容量 | ~5-10MB | 遍历耗时与键数量线性相关，典型场景 <1ms |
| 页面刷新 | 全页重载 | 无状态保留，这是预期行为 |
| 单键移除异常 | 某些键可能被第三方库监听 | try/catch 静默忽略，不阻断流程 |
