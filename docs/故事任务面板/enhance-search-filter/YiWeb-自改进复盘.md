# YiWeb-自改进复盘

> **版本**：1.0.0 · **复盘日期**：2026-05-22

## 观察

### O1 — 三面板筛选逻辑高度相似但分散实现

Claude 和 Story 面板的筛选逻辑几乎相同（搜索 + 多维标签 + 排序），但各自独立实现。AICR 的文件树筛选逻辑也在 fileTree 组件内闭包实现。

**影响**：未来新增面板或统一调整筛选行为时需修改多处，容易遗漏。

### O2 — 防抖时间不统一

实施前 fileTree 搜索防抖 200ms，searchMethods 300ms。统一为 300ms 后一致性好。

### O3 — 筛选状态管理方式不统一

- AICR：筛选状态在 `store`（vueRef）中
- Claude：筛选状态在组件 `data()` 中
- Story：筛选状态在组件 `data()` 中

## 诊断

| ID | 诊断 | 模式 | 等级 |
|----|------|------|------|
| D1 | 缺少共享筛选工具函数（filterByQuery、filterByTags、sortByField） | 代码重复 | P2 |
| D2 | 缺少通用筛选标签组组件（FilterTagGroup） | 组件复用 | P2 |
| D3 | 缺少通用排序控件组件（SortControl） | 组件复用 | P2 |

## 改进提案

### P1 — 提取共享筛选工具函数

将 `filterByQuery`、`filterByTags`、`sortByField` 提取到 `cdn/utils/filterUtils.js`，三个面板引用同一实现。

**预期效果**：减少代码重复，统一筛选行为。

### P2 — 提取通用筛选 UI 组件

将筛选标签组和排序控件提取为 CDN 通用组件，减少模板重复。

**预期效果**：新增面板时可快速复用。
