# YiWeb-使用场景

> **基线类型**：用户空间 · **版本**：1.0.0 · **生成日期**：2026-05-22

## 概述

描述用户如何使用搜索与过滤功能在三个面板中快速定位目标数据。覆盖正常路径、空状态路径、错误恢复路径。

### 主要价值

- 🧭 **导航效率** — 用户无需逐条翻页或滚动，通过搜索+筛选直达目标
- 🔬 **精准定位** — 多条件组合筛选在大量数据中缩小范围至个位数结果
- 🔄 **即时恢复** — 一键清除所有条件回到初始状态，无迷失风险
- 📋 **灵活排序** — 按时间/名称/数量等维度切换视角，满足不同查找意图

## §1 场景

### 场景 1：AICR 面板中查找特定文件

**用户角色**：代码审查者  
**目标**：在数百个文件组成的目录树中找到包含特定关键词的文件

```mermaid
flowchart TD
    START["打开 AICR 面板<br/>文件树已加载"] --> SEARCH["在搜索框输入关键词"]
    SEARCH --> DEBOUNCE["300ms 防抖等待"]
    DEBOUNCE --> FILTER["文件树按名称/路径匹配"]
    FILTER --> HAS_RESULT{"有匹配结果?"}
    HAS_RESULT -->|"是"| SHOW["显示匹配文件+父目录<br/>匹配文本高亮"]
    HAS_RESULT -->|"否"| EMPTY["显示空状态提示<br/>'未找到匹配文件'"]
    SHOW --> TAG["可选：点击标签按钮<br/>进一步缩小范围"]
    TAG --> COMBINED["搜索+标签组合结果"]
    EMPTY --> CLEAR1["点击清除或修改关键词"]
    COMBINED --> DONE["点击目标文件<br/>打开查看内容"]
    CLEAR1 --> SEARCH
```

| 路径 | 触发 | 预期 |
|------|------|------|
| 正常 | 输入匹配关键词 | 树实时过滤，匹配高亮，点击文件打开 |
| 空状态 | 输入无匹配关键词 | 显示"未找到匹配文件"提示 + 建议修改关键词 |
| 错误恢复 | 输入错误关键词 → 修改 | 每次输入重新过滤，实时反映 |

### 场景 2：AICR 面板中按标签筛选会话

**用户角色**：代码审查者  
**目标**：快速定位属于特定项目/模块的会话记录

```mermaid
flowchart TD
    START["AICR 面板<br/>会话列表已加载"] --> VIEW["查看标签筛选区"]
    VIEW --> SELECT["点击目标标签按钮"]
    SELECT --> ACTIVE["标签高亮为选中态"]
    ACTIVE --> FILTER["会话列表实时过滤<br/>显示含该标签的会话"]
    FILTER --> ADD["点击第二个标签"]
    ADD --> OR_FILTER["OR 逻辑：显示含任一<br/>选中标签的会话"]
    OR_FILTER --> SORT["可选：切换排序<br/>按时间/按名称"]
    SORT --> CLICK["点击目标会话<br/>进入对话"]
```

| 路径 | 触发 | 预期 |
|------|------|------|
| 正常 | 选中标签 → 结果过滤 | 列表仅显示匹配会话，标签计数实时更新 |
| 空状态 | 标签无对应会话 | 显示"无匹配会话"提示 |
| 多标签 | 选中多个标签 | OR 逻辑，结果范围扩大 |
| 清除 | 点击清除按钮 | 所有标签取消选中，列表恢复完整 |

### 场景 3：Claude 面板中查找配置完备度不足的项目

**用户角色**：项目管理者  
**目标**：找出缺少 CLAUDE.md 或 settings 等关键配置的项目

```mermaid
flowchart TD
    START["打开 Claude 面板<br/>项目列表已加载"] --> BROWSE["浏览统计栏<br/>了解全局配置覆盖情况"]
    BROWSE --> FILTER["点击健康状态筛选<br/>选择'含 CLAUDE.md'"]
    FILTER --> SHOW["卡片网格更新<br/>仅显示有此配置的项目"]
    SHOW --> ADD["追加选择'含 settings'"]
    ADD --> AND_FILTER["AND 逻辑：同时满足<br/>两项条件的项目"]
    AND_FILTER --> SORT["切换排序：按文件数降序"]
    SORT --> IDENTIFY["识别出关键项目<br/>点击查看详情"]
```

| 路径 | 触发 | 预期 |
|------|------|------|
| 正常 | 筛选条件匹配 | 卡片网格实时过滤 |
| 空状态 | 无项目满足组合条件 | "没有匹配的项目" + 建议调整筛选 |
| 多维度 | 叠加多个筛选 | AND 逻辑，结果逐级缩小 |
| 清除 | 点击清除或取消勾选 | 返回全量项目列表 |

### 场景 4：故事面板中按状态和类型定位故事

**用户角色**：项目参与者  
**目标**：快速查看"进行中的前端故事"以了解开发进度

```mermaid
flowchart TD
    START["打开故事面板<br/>故事列表已加载"] --> STATUS["点击状态筛选标签<br/>选择'编码进行中'"]
    STATUS --> FILTER1["列表/看板过滤<br/>仅显示该状态故事"]
    FILTER1 --> TYPE["点击类型筛选标签<br/>选择'前端'"]
    TYPE --> FILTER2["叠加 AND 过滤<br/>显示编码进行中的前端故事"]
    FILTER2 --> SEARCH["可选：搜索框输入关键词"]
    SEARCH --> FINAL["三重条件叠加结果"]
    FINAL --> SORT["列表视图：点击列头<br/>按修改时间降序"]
    SORT --> CLICK["点击目标故事<br/>查看详情"]
```

| 路径 | 触发 | 预期 |
|------|------|------|
| 正常 | 状态+类型+搜索组合 | 精确匹配目标故事集 |
| 空状态 | 组合条件无匹配 | "没有匹配的故事" + 清除筛选按钮 |
| 切换视图 | 看板→卡片→列表切换 | 筛选条件跨视图保持 |
| 清除 | Escape 或清除按钮 | 所有条件重置 |

---

### 场景 5：空状态恢复

**用户角色**：任意面板使用者  
**目标**：筛选无结果后快速恢复

```mermaid
flowchart TD
    FILTER["设置了筛选条件"] --> CHECK{"有结果吗?"}
    CHECK -->|"否"| EMPTY["显示空状态组件"]
    EMPTY --> OPTIONS["提供操作选项"]
    OPTIONS --> CLEAR["点击'清除筛选'按钮"]
    OPTIONS --> ESC["按 Escape 键"]
    OPTIONS --> MODIFY["修改搜索关键词"]
    CLEAR --> RESTORE["恢复默认显示"]
    ESC --> RESTORE
    MODIFY --> CHECK
```
