# YiWeb-故事任务

> **基线类型**：问题空间 · **版本**：1.0.0 · **生成日期**：2026-05-22

## 概述

当前三个面板（代码审查、Claude 管理、故事任务）的搜索与过滤能力薄弱：大部分仅支持按名称搜索，缺少状态筛选、日期范围、多条件组合、排序切换等功能。用户在面对大量数据时定位目标困难，交互效率低。

### 主要价值

- 🔍 **查找效率** — 用户可在 3 秒内定位目标文件/会话/项目/故事，无需手动滚动浏览
- 🏷️ **多维筛选** — 支持按状态、类型、日期、标签等多维度组合筛选，精准缩小范围
- ⚡ **即时反馈** — 搜索输入即时响应，结果显示高亮匹配文本，清空一键恢复
- 📊 **排序灵活** — 列表视图支持按名称、日期、数量等多字段升降序排序
- 🎯 **空状态引导** — 无匹配结果时给出清晰提示与操作建议，而非空白页面

### 业务规则

| R# | 描述 | 校验方式 | 证据级别 |
|----|------|---------|---------|
| R1 | 搜索防抖统一 300ms，避免频繁重渲染 | 检查各面板搜索输入处理 | B |
| R2 | 标签筛选为 OR 逻辑（选中任一即匹配），跨维度筛选为 AND 逻辑（搜索 + 标签 + 状态叠加取交集） | 检查筛选 computed 逻辑 | B |
| R3 | 空搜索/空筛选等同于"显示全部"，不清空已有数据 | 检查边界处理 | B |
| R4 | 排序状态应有视觉指示（箭头图标），默认排序为按时间降序 | 检查排序 UI 组件 | B |
| R5 | 所有搜索/筛选输入框支持 Escape 键清除 | 检查键盘事件处理 | B |

### 数据约束

| 约束 | 类型 | 范围/格式 | 来源 |
|------|------|----------|------|
| 搜索关键词 | string | 任意文本，trim 后匹配 | 用户输入 |
| 防抖延迟 | number | 300ms | 交互规范 |
| 标签选择 | string[] | 现有标签集合的子集 | 数据源 |
| 排序字段 | enum | name / updatedAt / fileCount / skillCount 等 | 数据模型 |
| 排序方向 | enum | asc / desc | 交互约定 |
