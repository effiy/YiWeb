## §1 使用场景

| 维度 | 内容 |
|------|------|
| **角色** | 接到开发任务的功能开发者 |
| **前置** | 需要新增一个功能模块，不确定应放在哪一层 |
| **操作流** | 确认功能性质 → 查分层决策表 → 判断归属层级 → 在对应目录创建文件 → 遵循 import 方向约束 |
| **后置** | 功能归属正确，import 方向符合上层依赖下层约束 |
| **异常** | 功能跨越多层 → 按主要职责判定归属，跨层依赖走正常 import |

```mermaid
flowchart TD
    A["接到新功能需求"] --> B{"功能是什么类型？"}
    B -->|"页面交互逻辑"| C["L1 视图层"]
    B -->|"API/认证/业务"| D["L2 服务层"]
    B -->|"工具/组件/渲染"| E["L3 基础设施层"]
    B -->|"HTML 模板"| F["L0 展示层"]
    C --> G["src/views/&lt;name&gt;/"]
    D --> H["src/core/services/"]
    E --> I["cdn/utils/"]
    F --> J["src/views/&lt;name&gt;/index.html"]
```

## §2 技术评审

| 评审项 | 结论 | 说明 |
|--------|------|------|
| 决策表完整性 | 通过 | 覆盖全部 4 层的功能类型判定规则 |
| 决策准确性 | 通过 | 现有全部模块归属与决策表一致 |
| 边界模糊处理 | 通过 | 跨层功能按主要职责判定，有明确处理规则 |


```mermaid
flowchart TB
    Q{"新增功能\n属于哪层？"}
    Q -->|"页面/UI 交互"| L1["L1 视图层\nsrc/views/"]
    Q -->|"API 封装/数据操作"| L2["L2 服务层\nsrc/core/"]
    Q -->|"通用工具/组件"| L3["L3 基础设施层\ncdn/"]
    L1 -.-> CHECK1["✅ 可依赖 L2/L3"]
    L2 -.-> CHECK2["✅ 可依赖 L3\n❌ 不可依赖 L1"]
    L3 -.-> CHECK3["❌ 不可依赖 L1/L2"]
```
