## §2 测试用例

### 2.1 正常用例 (TC-N*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-N01 | AICR 页面已加载，标签行包含超出视口宽度的标签（通过缩小浏览器窗口或添加大量标签触发），出现水平滚动条 | 用户向右水平滚动标签行（鼠标拖拽或触控板横向滑动） | 筛选按钮（`.aicr-meta-filter-actions`）保持在标签行可视区域左侧固定不动，不随标签内容水平滚动；筛选按钮完全可见，未被任何元素遮挡 | FP1 | P0 |
| TC-N02 | 标签行处于水平滚动偏移状态（非最左侧） | 用户点击筛选按钮（`.aicr-meta-filter-toggle`） | 筛选按钮可正常点击交互，筛选面板正常展开；筛选按钮区域背景完整遮盖后方滚动的标签内容，无内容透出 | FP1 | P0 |
| TC-N03 | AICR 页面已加载，标签行宽度在视口范围内，无水平滚动条 | 用户观察筛选按钮位置 | 筛选按钮正常显示在标签行首位，与右侧标签间距适度（padding-right: 8px + margin-right: 2px）；按钮外观与变更前一致（尺寸、颜色、圆角、字体） | FP1 | P1 |

### 2.2 边界用例 (TC-B*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-B01 | 标签行仅比视口宽 1 个像素，出现极细的水平滚动条 | 用户做极小幅度的水平滚动 | 筛选按钮仍 sticky 固定在左侧，不出现抖动或闪烁；sticky 附着的阈值正确触发 | FP1 | P1 |
| TC-B02 | 标签行极宽（如 100+ 个标签） | 用户快速拖拽滚动条到最右侧 | 筛选按钮始终固定在左侧；背景色不因大量标签滚动产生视觉瑕疵 | FP1 | P1 |
| TC-B03 | 浏览器窗口大小改变导致标签行从非超宽变为超宽（或反之） | 用户拖拽浏览器窗口边缘调整宽度 | sticky 行为和 inline-flex 行为之间切换无缝，无视觉跳变；筛选按钮位置始终合理 | FP1 | P1 |

### 2.3 异常用例 (TC-E*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-E01 | 使用不支持 CSS `position: sticky` 的旧浏览器（如 IE 11） | 用户水平滚动标签行 | 筛选按钮随标签一起滚动（退化为 static 定位），筛选功能仍可正常使用；页面不崩溃，无 JavaScript 错误 | FP1 | P1 |
| TC-E02 | CSS 变量 `--yi-bg` 因主题切换或其他原因未定义或为 `transparent` | 用户水平滚动标签行 | 筛选按钮区域背景可能透明，后方滚动标签内容可见但不影响筛选按钮的交互功能；不出现 JavaScript 错误 | FP1 | P2 |

### 2.4 回归用例 (TC-R*)

| ID | Given | When | Then | 关联 FP | 优先级 |
|----|-------|------|------|---------|--------|
| TC-R01 | 标签行在视口范围内，无水平滚动 | 用户依次执行：点击筛选按钮 → 选择一个过滤标签 → 点击清除按钮 → 点击筛选按钮关闭面板 | 所有筛选交互功能正常：面板展开/收起、标签过滤/清除与变更前行为完全一致；`.aicr-meta-filter-toggle` 和 `.aicr-meta-filter-clear` 的样式（悬停效果、颜色变化）不变 | FP1 | P0 |
| TC-R02 | 标签行超宽并水平滚动到中间位置 | 用户观察筛选按钮与 `.aicr-tags-fade--left` 的相对层级 | 筛选按钮（z-index: 3）完全位于左侧 fade 渐变遮罩（z-index: 2）之上；fade 遮罩不可覆盖或遮挡筛选按钮区域 | FP1 | P0 |
