## §1 正常路径

### TC-N01: Token 自动注入
| Given | Token 已存储 | When | 调用 getData('/api/sessions') | Then | 请求头包含 `X-Token` |

### TC-N02: 401 自动处理
| Given | Token 无效 | When | API 返回 401 | Then | Token 清除 + 登录弹窗 |

### TC-N03: Token 过期自动清除
| Given | Token 已过期 | When | 调用 hasValidToken() | Then | 返回 false，getStoredToken 返回 null |

### TC-N04: 请求去重
| Given | 正在进行相同请求 | When | 再次调用 | Then | 返回同一个 Promise，不重复发送 |

### TC-N05: 缓存命中
| Given | 已缓存 `/api/projects` 结果（未过期） | When | 再次调用 getData('/api/projects') | Then | 直接返回缓存数据 |

### TC-N06: 流式请求正常
| Given | API 返回 SSE 流 | When | 调用 streamPrompt | Then | onChunk 逐块接收，最终返回完整文本 |

## §2 边界/异常

### TC-E01: 401 多次触发防抖
| Given | Token 无效 | When | 短时间内多次 401 | Then | 仅触发一次登录弹窗（2 秒冷却） |

### TC-E02: 网络错误
| Given | 网络断开 | When | 调用任意 API | Then | 错误对象含 message/status/url/timestamp |

### TC-E03: 流式中断
| Given | SSE 连接中断 | When | 流式传输过程中网络断开 | Then | 返回已接收的部分文本 + 错误提示 |

### TC-E04: 缓存过期
| Given | 缓存 TTL 已过 | When | 调用 getData | Then | 重新请求 API 并更新缓存 |

### TC-E05: 批量请求部分失败
| Given | 批量 5 个请求，其中 1 个失败 | When | batchRequests | Then | results 含 4 个成功，errors 含 1 个失败 |

### TC-E06: 重试耗尽
| Given | API 持续返回 500 | When | retryRequest maxRetries=3 | Then | 3 次重试后抛出错误 |
