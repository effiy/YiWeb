## §1 STRIDE 威胁建模

### S — Spoofing（身份伪造）

| 威胁 | 缓解 |
|------|------|
| 伪造 X-Token 冒充用户 | Token 由后端签发 JWT，前端仅透传 |
| 中间人修改请求 | fetch 通过 HTTPS 发送（生产环境 API 为 `https://api.effiy.cn`） |
| 伪造请求来源 | 后端校验 Origin/Referer |

### T — Tampering（数据篡改）

| 威胁 | 缓解 |
|------|------|
| 篡改请求参数 | 参数验证层（required/string/number/email/url） |
| 拦截并修改响应 | HTTPS 传输加密 |
| 篡改 localStorage Token | 仅影响客户端，后端校验 Token 有效性 |

### R — Repudiation（抵赖）

| 威胁 | 缓解 |
|------|------|
| 无操作审计 | logInfo 记录关键操作；后端 API 记录操作者 |
| 无法追溯请求来源 | 请求时间戳自动注入 |

### I — Information Disclosure（信息泄漏）

| 威胁 | 缓解 |
|------|------|
| Token 明文存储 localStorage | Token 为 JWT，有过期时间 |
| API 响应缓存泄漏 | 缓存仅在内存中，页面关闭即清除 |
| 日志输出敏感数据 | Structured logging，不记录 Token 值 |
| 请求 URL 暴露查询参数 | HTTPS 加密传输 |
| 错误堆栈暴露内部路径 | 生产环境 debug=false |

### D — Denial of Service（拒绝服务）

| 威胁 | 缓解 |
|------|------|
| 并发请求洪水 | 请求去重 + 批量合并 |
| 缓存膨胀 | 最大 100 条目 + TTL 自动清理 |
| 流式连接耗尽 | 浏览器原生连接池限制 |
| AbortController 泄漏 | 请求完成/失败自动清理 |

### E — Elevation of Privilege（权限提升）

| 威胁 | 缓解 |
|------|------|
| 修改 localStorage env 切换环境获得权限 | env 切换仅改变 URL，Token 不变 |
| 修改 localStorage 绕过认证 | 后端校验 Token 有效性 |
| 通过 401 重试逻辑绕过 | 轮询 Token 时有最大等待时间（3s） |

## §2 认证面审计

| 检查点 | 实现 | 状态 |
|------|------|:--:|
| Token 存储 | localStorage `YiWeb.apiToken.v1` | ✅ |
| Token 传输 | `X-Token` 请求头 | ✅ |
| Token 清除 | `clearToken()` 移除 localStorage | ✅ |
| 401 处理 | `handle401Error()` 清除 + 弹窗 | ✅ |
| 凭证隔离 | `credentials: 'omit'` | ✅ |
| Token 校验 | `hasValidToken()` 存在性检查 | ⚠️ 仅检查存在，不解码 |
| 登录弹窗 | `<dialog>` + token 输入 | ✅ |
| 暴力破解保护 | 401 处理 2 秒冷却 | ✅ |
