# 故事任务

> **基线类型**：问题空间 · **版本**：1.0.0 · **生成日期**：2026-05-26

## 概述

核心服务层是 YiWeb 的前后端桥梁，负责 API 请求、认证管理、错误处理、数据缓存和业务分析。所有视图通过统一的 `src/core/services/` 模块访问后端能力，确保一致的认证、缓存和错误处理行为。

### 主要价值

- 🔗 **统一 API 层** — 所有视图使用同一套 fetch 封装，保证一致的请求格式和错误处理
- 🔐 **认证管理** — X-Token 生命周期管理 + 401 自动拦截 + 登录弹窗
- ⚡ **性能优化** — 内存缓存 + 请求去重 + 批量操作 + 防抖节流
- 📊 **业务分析** — 业务流程建模和场景分析能力
- 🔄 **会话同步** — 文件系统与后端会话双向同步

## §2 Requirements

### 功能点

| FP# | 描述 | 优先级 |
|-----|------|------|
| FP1 | 环境配置 — local/prod 双环境切换，URL 构建 | P0 |
| FP2 | HTTP 请求 — GET/POST/PUT/PATCH/DELETE + 超时/拦截器 | P0 |
| FP3 | 认证注入 — 自动注入 X-Token 请求头 | P0 |
| FP4 | 401 处理 — 自动清除 Token + 弹出登录弹窗 | P0 |
| FP5 | Token 管理 — 存储/读取/清除/验证 + 模型选择 | P0 |
| FP6 | 请求缓存 — 内存缓存 + TTL + 强制刷新 | P1 |
| FP7 | 批量请求 — 并发的批量请求 + 结果收集 | P1 |
| FP8 | 自动重试 — 指数退避重试 | P1 |
| FP9 | 流式请求 — SSE 流式响应 + JSON 降级 | P1 |
| FP10 | CRUD 封装 — 带验证的增删改查 + 缓存清空 | P0 |
| FP11 | 数据验证 — required/string/number/email/url/custom | P1 |
| FP12 | 会话同步 — 文件 ↔ 会话双向同步 + 队列批处理 | P1 |
| FP13 | 业务建模 — 流程识别/分析和场景派生 | P2 |
| FP14 | URL/网络工具 — query string 构建、online 检测 | P1 |

### 业务规则

| R# | 描述 |
|----|------|
| R1 | 所有请求显式设置 `credentials: 'omit'` 避免携带 Cookie |
| R2 | Token 通过 `X-Token` 请求头传递，存储在 localStorage |
| R3 | 401 错误自动清除 Token 并触发登录弹窗，2 秒冷却防重复 |
| R4 | 缓存 TTL 默认 5 分钟，最大 100 条目 |
| R5 | 请求超时默认 5 分钟，通过 AbortController 控制 |

## §4 AC

| AC# | Given | When | Then |
|-----|-------|------|------|
| AC1 | 用户已登录并持有 Token | 任何 API 请求 | X-Token 自动注入请求头 |
| AC2 | API 返回 401 | 请求被拦截 | Token 清除 + 登录弹窗 |
| AC3 | Token 过期 | 页面加载 | 自动清除 + 提示登录 |
| AC4 | 相同请求并发 | 短时间内多次调用 | 请求去重，仅发一次 |
| AC5 | 网络错误 | fetch 抛出异常 | 格式化的错误对象 |
| AC6 | 缓存有效期内的请求 | 调用 getData | 返回缓存数据，不发请求 |
