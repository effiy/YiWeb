## §1 STRIDE 威胁建模

### S — Spoofing | T — Tampering

| 威胁 | 缓解 |
|------|------|
| 组件名称伪造 | ComponentRegistry 从可信 CDN 路径加载 |
| 组件注入 | 组件路径在 createBaseView 中显式声明 |
| localStorage 篡改影响组件行为 | 仅 HeaderActions 读取（动画/缓存设置），无安全影响 |

### I — Information Disclosure

| 威胁 | 缓解 |
|------|------|
| Markdown 内容 XSS | **SanitizePlugin** 过滤所有 HTML 标签和属性 |
| Mermaid 图表 SVG XSS | Mermaid 生成的 SVG 不含脚本 |
| localStorage 存储敏感数据 | storage 工具仅存储配置项，不存敏感信息 |
| 日志输出泄漏 | logInfo 在生产环境受限 |

### R — Repudiation

| 威胁 | 缓解 |
|------|------|
| 无 UI 操作审计 | 组件为展示层，不涉及审计 |

### D — Denial of Service

| 威胁 | 缓解 |
|------|------|
| 超大 Markdown 导致客户端假死 | marked 解析在微任务，不阻塞主线程；超长内容拆段 |
| 递归 Mermaid 图表 | Mermaid 渲染有超时保护 |
| CSS 文件无限循环 @import | 样式文件手动管理，无动态注入 |

### E — Elevation of Privilege

| 威胁 | 缓解 |
|------|------|
| 通过 v-html 注入执行脚本 | SanitizePlugin 移除 `<script>`、`onerror`、`onload` 等 |
| 通过 Markdown 链接钓鱼 | 链接渲染为 `<a>` 标签，用户需手动点击 |

## §2 v-html 审计

| 使用点 | 内容来源 | 风险 | 缓解 |
|------|------|:--:|------|
| MarkdownView | 用户/API 提供的 Markdown | 高 | SanitizePlugin |
| CodeView (语法高亮) | 文件内容 | 中 | 自研高亮不执行 HTML |
| YiModal body | 组件模板控制 | 低 | 模板由开发者控制 |
| YiLoading text | 组件 prop 控制 | 低 | prop 由开发者传入 |

---

## §3 SanitizePlugin 规则

```
允许的标签: h1-h6, p, div, span, a, img, table, thead, tbody, tr, th, td,
            ul, ol, li, pre, code, blockquote, em, strong, del, br, hr, svg, path
允许的属性: href, src, alt, title, class, id, target, rel, width, height
禁止: <script>, <iframe>, <object>, <embed>, on* attributes, javascript: URLs
```
