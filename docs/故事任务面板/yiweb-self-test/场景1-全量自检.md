## §1 使用场景

| 维度 | 内容 |
|------|------|
| **角色** | 项目管理者 |
| **前置** | 项目已通过初始化工具完成初始化，项目根目录存在项目说明文件 |
| **操作流** | 扫描项目根目录 → 检查项目说明(完整项目画像?) → 检查项目说明书(≥3个领域术语?) → 检查故事目录(每个任务都有完整文档?) → 检查架构规则(模块间有违规引用?) → 检查安全(代码中藏有密码或密钥?) → 检查版本(各处版本号一致?) → 检查分支(分支名规范?) → 全部通过生成健康报告 |
| **后置** | 生成完整健康报告，任意一项失败则管道挂起等待修复 |
| **异常** | 检查工具不可用 → 降级模式，跳过依赖不可用工具的检查项 |

```mermaid
flowchart LR
    A["扫描项目根目录"] --> B["检查项目说明"]
    B --> C["检查项目说明书"]
    C --> D["检查故事目录"]
    D --> E["检查架构规则"]
    E --> F["检查安全"]
    F --> G["检查版本一致性"]
    G --> H["检查分支规范"]
    H --> I{"全部通过?"}
    I -->|"是"| J["生成健康报告"]
    I -->|"否"| K["挂起管道等待修复"]
```

## §2 技术评审

| 评审项 | 结论 | 说明 |
|--------|------|------|
| 检查项覆盖完整性 | 通过 | 15 项检查覆盖文档/结构/安全/分支/版本 |
| 执行模式设计 | 通过 | 顺序/并行/降级三种模式 |
| 性能目标 | 通过 | 全量自检 ≤ 2 秒 |

### 检查项注册表

| ID | 类别 | 优先级 | 阻断 | 检查内容 |
|----|------|:---:|:---:|------|
| claude-completeness | document | P0 | 是 | 项目说明是否包含五大标记段 |
| readme-domain | document | P0 | 是 | 项目说明书是否包含 ≥3 个领域术语 |
| story-structure | structure | P0 | 是 | 每个故事目录是否包含完整基线文档 |
| story-fmeta | document | P0 | 是 | 每个文档是否包含完整版头 |
| story-nav | document | P0 | 是 | 文档导航是否对齐且链接有效 |
| story-changelog | document | P0 | 是 | 每个文档底部是否包含变更记录表 |
| cross-ref-valid | document | P0 | 是 | 文档内相对链接指向的文件是否存在 |
| stale-ref | document | P1 | 否 | 文档引用的外部文件版本号是否过期 |
| branch-isolation | branch | P0 | 是 | 当前分支是否为规范格式且从 main 分出 |
| version-consistency | version | P0 | 是 | 各处版本号是否一致 |
| arch-layer | structure | P0 | 是 | 视图层是否违规直接引用基础设施层 |
| security-no-secrets | security | P0 | 是 | 代码中无硬编码密钥或密码 |
| security-fetch-credentials | security | P0 | 是 | 所有网络请求是否设置安全凭证模式 |
| security-no-raw-log | security | P1 | 否 | 无裸日志调用 |
| config-env | structure | P0 | 是 | 环境配置仅支持规定的两种环境名 |

### 执行模式

| 模式 | 适用场景 | 策略 |
|------|---------|------|
| 顺序模式 | Gate A 前置检查 | 按注册表顺序执行，遇阻断立即中断 |
| 并行模式 | 全量自检 | 同类别内并发执行 |
| 降级模式 | 部分工具不可用 | 跳过不可用检查项，标记原因 |
