# 故事任务

> **基线类型**：问题空间 · **版本**：1.0.0 · **生成日期**：2026-05-26

## 概述

故事任务面板是 YiWeb 的任务管理视图，用于展示和跟踪项目的所有故事（Story）及其文档。面板从远端 API 获取故事数据，支持列表/看板/卡片三种视图模式，按状态、类型、项目等多维度筛选和排序，帮助团队掌握项目全貌。

### 主要价值

- 📊 **故事全景** — 统一查看所有故事及其状态、类型、关联文档
- 🔍 **多维筛选** — 按状态、类型、项目标签组合筛选
- 📋 **灵活视图** — 列表、看板、卡片三种展示模式
- 📝 **文档关联** — 每个故事关联基线文档（故事任务/使用场景/技术评审/安全审计/测试设计）

## §1 Story

### Story 1: 故事列表展示

| 字段 | 内容 |
|------|------|
| 作为 | 项目管理者 |
| 我想要 | 查看所有故事及其状态概览 |
| 以便 | 掌握项目进度和任务分布 |
| 优先级 | P0 |
| 范围边界 | 故事查询 + 列表/卡片渲染 |

#### §1.1 User Operations

| # | 操作 | 触发条件 | 操作步骤 | 预期结果 |
|---|------|---------|---------|---------|
| 1 | 故事列表加载 | 页面首次加载 | 从远端 API 获取故事数据 | 列表/卡片展示故事 |
| 2 | 切换视图 | 点击视图切换按钮 | 列表 ↔ 看板 ↔ 卡片 | 展示模式切换 |
| 3 | 点击故事 | 点击故事行/卡片 | 选中故事 | 详情卡片展示 |
| 4 | 搜索故事 | 输入关键词 | 300ms 防抖 → 按名称/描述匹配 | 实时过滤 |

### Story 2: 故事筛选与排序

| 字段 | 内容 |
|------|------|
| 作为 | 项目管理者 |
| 我想要 | 按状态、类型、项目筛选和排序故事 |
| 以便 | 快速定位特定任务并了解分布 |
| 优先级 | P1 |
| 依赖 | Story 1 |

#### §1.1 User Operations

| # | 操作 | 触发条件 | 操作步骤 | 预期结果 |
|---|------|---------|---------|---------|
| 1 | 状态筛选 | 点击状态标签 | 选择 pending/in_progress/completed | 仅显示对应状态 |
| 2 | 类型筛选 | 点击类型标签 | 选择前端/后端/全栈/元数据 | 仅显示对应类型 |
| 3 | 项目标签筛选 | 点击项目标签 | 选择项目 | 仅显示对应项目 |
| 4 | 组合筛选 | 同时应用多条件 | 多条件叠加取交集 | 精确结果 |
| 5 | 列表排序 | 点击列头 | 切换该列升降序 | 列表重排 |
| 6 | 清除筛选 | 点击清除或 Escape | 所有条件重置 | 显示全部 |

## §2 Requirements

### 功能点

| FP# | 描述 | 输入 | 输出 | 错误行为 | 优先级 |
|-----|------|------|------|---------|--------|
| FP1 | 故事数据加载 — 从 API 查询故事列表 | API 请求 | 故事数据数组 | API 失败显示错误状态 | P0 |
| FP2 | 状态判定 — 根据文档存在性判定故事阶段 | 故事数据 | 状态标签 | 缺失文档判定为 incomplete | P0 |
| FP3 | 列表渲染 — 表格形式展示故事 | 故事数组 | 表格 DOM | 无数据显示空状态 | P0 |
| FP4 | 卡片渲染 — 卡片形式展示故事 | 故事数组 | 卡片 DOM | 无数据显示空状态 | P1 |
| FP5 | 搜索过滤 — 按名称/描述搜索 | 搜索词 | 过滤后故事 | 空搜索显示全部 | P0 |
| FP6 | 状态筛选 — 按状态过滤 | 状态值 | 过滤后故事 | 无筛选显示全部 | P1 |
| FP7 | 类型筛选 — 按类型过滤 | 类型值 | 过滤后故事 | 无筛选显示全部 | P1 |
| FP8 | 列表排序 — 按列排序 | 排序列 + 方向 | 排序后故事 | 默认时间降序 | P1 |
| FP9 | 详情卡片 — 查看故事完整信息 | 故事名 | 详情展示 | 加载失败占位 | P1 |

### 业务规则

| R# | 描述 |
|----|------|
| R1 | 故事数据来自远端 API，不读本地文件系统 |
| R2 | 面板仅展示和查看，不修改故事数据 |
| R3 | 状态由文档存在性判定（baseline documents present → stage） |
| R4 | 多筛选条件为 AND 逻辑叠加 |

## §3 成功标准

| SC# | 描述 | 度量方式 | 目标值 | 优先级 |
|-----|------|---------|--------|--------|

## §4 范围边界

### 范围内
- 故事列表/卡片展示
- 状态、类型、项目标签筛选
- 搜索过滤 + 排序
- 故事详情卡片

### 范围外
- 故事创建/编辑/删除（由 /rui 流程管理）
- 文档内容编辑
- 故事依赖关系可视化
