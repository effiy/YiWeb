## 概述

故事任务面板是 YiWeb 的项目管理视图，用于展示和跟踪所有故事的完整生命周期。用户在一个面板中完成：浏览故事列表/看板/卡片、按状态/类型/项目多维筛选、查看故事详情与关联文档。

面向的用户角色：项目管理者、开发者、团队新成员。

**数据来源**：面板数据全部来自远端 API（`POST API_URL/`，`query_documents` 方法），前端不在本地存储或生成故事数据。数据流：API 请求 → `fetchStories()` 拉取全量文档 → 按文件路径第二段（故事名称）分组 → `determineStatus()` 基于文件名后缀判定阶段 → 计算健康分/项目标签/下一步等元数据 → `stories` ref 响应式数组 → computed 分组 → 三视图渲染。筛选为纯前端计算，不触发后端请求。

### 主要价值

- 📊 **故事全景** — 统一查看所有故事及其状态、类型、关联文档，掌握项目全貌
- 🔍 **多维筛选** — 按状态、类型、项目标签组合筛选，快速定位目标故事
- 📋 **灵活视图** — 列表、看板、卡片三种展示模式，适应不同浏览场景
- 📝 **文档关联** — 每个故事关联基线文档（故事任务/使用场景/技术评审/安全审计/测试设计），阶段一目了然
- 🔗 **审查联动** — 点击故事文件可直接跳转到 AICR 审查面板查看文档内容
- 🚫 **明确的不做清单** — 不做故事创建/编辑/删除、不做依赖关系可视化、不做文档内容在线编辑

### Story 3: 标签筛选与搜索

作为项目管理者，我想要通过项目标签栏和关键词搜索快速定位故事，以便在海量故事中即时找到目标。优先级 P1。依赖 Story 1。

| 一键清除所有筛选 | 用户点击"清除"按钮或按 Escape 键 | 所有筛选条件（标签+搜索+Filter Controls）重置 | 完整故事列表恢复；筛选胶囊行消失；全部按钮高亮 |

#### 范围边界

**范围内**：关键词搜索（300ms 防抖）、项目标签栏水平滚动多选筛选、标签颜色 hash 编码、未分类故事筛选、活跃筛选胶囊展示与独立关闭、筛选条件一键清除、项目标签与故事标签双向级联。**范围外**：全文搜索（搜索文档内容）、正则搜索、标签创建/编辑/删除、标签层级管理（扁平标签模型）、筛选条件跨会话持久化。

## §2 范围边界

### 范围内（做什么）

| 条目 | 关联 FP# | 产品决策依据 |
|------|---------|-------------|
| 故事数据从远端 API 加载并展示 | FP1, FP2 | 面板需要反映最新的故事状态 |
| 看板/卡片/列表三种视图模式 | FP3, FP4, FP5 | 不同角色和场景需要不同的浏览方式 |
| 故事详情侧面板（描述/下一步/通知/日志/文件清单） | FP6 | 用户需要查看故事完整信息而不离开面板 |
| 故事名称/文档类型/缺失文档三维筛选 | FP7, FP8, FP9 | Filter Controls 面板提供精细化筛选能力 |
| 列排序（名称/状态/文件数/日期） | FP10 | 帮助用户按需组织列表 |
| Filter Controls 折叠 | FP11 | 减少筛选区占用空间，按需展开 |
| 关键词搜索（300ms 防抖） | FP12 | 快速定位已知名称的故事 |
| 项目标签栏多选筛选（水平滚动+颜色编码） | FP13, FP14 | 以项目维度快速缩小故事范围 |
| 未分类故事独立筛选 | FP15 | 快速识别无项目归属的孤立故事 |
| 活跃筛选胶囊摘要行（独立关闭+一键清除） | FP16, FP17 | 可视化当前筛选状态，精准移除单个条件 |
| 项目标签与故事标签双向级联 | BR1 | 保持标签选择的一致性 |
| 文件点击跳转 AICR 审查面板 | — | 从故事概览到文档详查的无缝衔接 |

### 范围外（不做什么）

| 条目 | 排除原因 | 产品决策 |
|------|---------|---------|
| **故事创建/编辑/删除** | 故事由 /rui 管线流程管理，面板仅展示 | 保持工具边界：管线负责写入，面板负责查看 |
| **文档内容在线编辑** | 编辑器是独立产品范畴，复杂度远超面板定位 | 编辑操作应在 AICR 审查面板中完成 |
| **故事依赖关系可视化** | 依赖图需要专门的图形渲染引擎 | 依赖信息在详情面板中以文本展示 |
| **多人实时协作**（同时查看、状态同步） | 需要 WebSocket 基础设施 | 故事面板是个人工作区 |
| **离线模式** | 面板依赖远端 API 提供故事数据 | 不投入资源支持离线场景 |
| **筛选条件持久化**（保存为书签或预设） | 筛选条件组合个性化，持久化管理增加复杂度 | 筛选操作足够轻量，无需跨会话记忆 |
| **全文搜索**（搜索文档内容而非仅故事名称） | 全文搜索需要索引构建和后端支持 | 文件名/故事名搜索覆盖主要定位场景 |
| **批量操作**（批量修改状态/标签） | 故事修改是管线职责 | 面板定位为只读看板 |
| **自定义看板列** | 阶段定义由文档基线固定 | 五阶段模型统一所有故事 |
| **故事对比**（并排比较两个故事） | 非核心使用场景 | 用户可通过详情面板逐个查看 |
| **导出为文件**（导出故事列表为 CSV/Excel） | 属于数据导出功能，非审查核心流程 | 未来可评估 |
| **正则表达式搜索** | 大多数用户不需要正则搜索能力 | 保持搜索简单直观 |

## §4 风险与假设

### 风险

- ⚠️ **故事数量过大导致性能下降** — 可能性：低 · 影响：中 · 缓解：当前 limit: 10000 全量加载，computed 缓存过滤结果；超过阈值时评估后端分页或虚拟滚动
- ⚠️ **API 故事数据不完整导致状态判定错误** — 可能性：中 · 影响：中 · 缓解：缺失字段默认 unknown；状态判定基于文件名后缀（结构化优先于约定）
- ⚠️ **项目标签提取依赖文件路径结构** — 可能性：中 · 影响：低 · 缓解：非标准路径结构的文件归入"未分类"；目录结构变更时需同步更新提取逻辑
- ⚠️ **标签数量过多导致标签栏性能下降** — 可能性：低 · 影响：低 · 缓解：水平滚动 + CSS 渐隐指示；超大量标签时评估虚拟化或搜索式标签选择器
- ⚠️ **标签颜色 hash 冲突导致相邻标签颜色相近** — 可能性：低 · 影响：低 · 缓解：hash 分布在 0–360 hue 空间；冲突时用户仍可通过标签文本区分
- ⚠️ **用户以为搜索是全文搜索而非故事名称搜索** — 可能性：中 · 影响：低 · 缓解：搜索框占位文本明确提示"搜索故事..."；搜索范围覆盖名称/描述/下一步
- ⚠️ **远端 API 不可用导致面板空白** — 可能性：低 · 影响：高 · 缓解：展示错误状态 + 重试按钮；不清除上次成功加载的缓存数据

### 假设

- 💡 **故事数量在可接受范围（< 500）** — 超过阈值时建议引入分页或虚拟滚动
- 💡 **用户的主要使用场景是查看和定位故事** — 产品方向基于此假设；如数据表明编辑需求强烈，重新评估边界
- 💡 **五阶段模型覆盖所有故事生命周期** — 阶段定义与文档基线一一对应；新增文档类型时需更新判定逻辑
- 💡 **文件命名遵循 `*-{文档类型}.md` 后缀约定** — 状态判定和文档类型识别基于后缀匹配（-故事任务.md / -使用场景.md / -实施报告.md / -测试报告.md / -自改进复盘.md），与 aicr 面板的标签提取逻辑一致
- 💡 **项目标签为扁平结构（非层级）** — 当前不引入标签父子关系或分组；如项目规模增长需要层级标签，重新评估
