## §1 使用场景

| 维度 | 内容 |
|------|------|
| **角色** | 需要新增一种数据处理链路的架构决策者 |
| **前置** | 现有三条数据流不足以覆盖新场景 |
| **操作流** | 分析新链路的起点和终点 → 识别链路经过的中间节点 → 标注每个节点的处理逻辑 → 与现有三条流对比是否有可复用的节点 → 绘制新流 mermaid 图 → 标注每节点的入口文件 → 补充到数据流文档 |
| **后置** | 新数据流已文档化，与现有流不重复 |
| **异常** | 新流与现有流高度重叠 → 考虑在现有流基础上标注分支而非新建 |

## §2 技术评审

| 评审项 | 结论 | 说明 |
|--------|------|------|
| 扩展方法论 | 通过 | 明确新增数据流的 7 步操作指南 |
| 复用现有流 | 通过 | 优先在现有流上标注分支 |
| 格式一致性 | 通过 | 新流遵循节点标注规范 |

### 新增数据流操作指南

| 步骤 | 操作 | 参考 |
|------|------|------|
| 1 | 明确新链路的起点和终点 | 现有三条流图的 mermaid 结构 |
| 2 | 确定起点到终点的中间处理步骤 | useMethods.js → crud.js → requestHelper.js 链 |
| 3 | 标注每节点的入口文件和状态变更点 | 节点标注规范（名称+文件+状态变更+常见问题） |
| 4 | 与现有三条流对比，是否有可复用的节点 | 命令流/加载流/文档流节点清单 |
| 5 | 绘制 mermaid 流程图 | flowchart LR 或 TD |
| 6 | 补充配套节点表 | 含入口文件+常见问题列 |
| 7 | 更新文档 | 补充到数据流文档 |
