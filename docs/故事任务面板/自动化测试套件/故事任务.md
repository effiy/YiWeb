# 自动化测试套件

> **Story ID**: yiweb-auto-test-suite
> **版本**: v1.0.0
> **状态**: 实现中
> **创建日期**: 2026-06-03

## 概述

为 YiWeb 项目搭建可执行的自动化测试体系，覆盖核心业务逻辑、API 接口、数据持久化、异常路径与边界、集成回归五大维度。基于 Vitest + jsdom，确保零构建链约束下的测试全覆盖。

## 功能点 (FP)

| # | 功能点 | 描述 | 优先级 |
|---|--------|------|:------:|
| FP1 | 核心业务逻辑测试 | 验证 `createBaseView` 视图工厂、store 响应式、computed 计算属性、methods 副作用 | P0 |
| FP2 | API 接口测试 | 验证 `requestHelper` 请求封装、`crud.js` 数据操作、认证头注入、401 处理 | P0 |
| FP3 | 数据持久化测试 | 验证 `localStorage` Token 存储、环境配置持久化、缓存管理 | P1 |
| FP4 | 异常路径与边界测试 | 验证空值/超时/网络错误/组件加载超时等边界场景 | P1 |
| FP5 | 集成与回归测试 | 验证 config→services→store→computed→DOM 全链路、跨模块协作 | P2 |

## 风险

| 风险 | 等级 | 缓解措施 |
|------|:---:|---------|
| ESM 模块在 jsdom 中导入路径解析失败 | 中 | vitest.config.js 已配置 alias 映射 `/cdn`, `/src`, `/tests` |
| 浏览器 API (Vue CDN) 在 Node 环境不可用 | 高 | mock 全局 Vue 对象；jsdom 提供 window/document 基础 API |
| 异步组件加载测试不稳定 | 中 | 使用 vitest fake timers + waitFor |
| 覆盖率统计因 ESM 动态导入不准确 | 低 | 明确标记排除项 |
