## §1 Story

**角色**: YiWeb 终端用户

**价值**: 当应用出现数据异常或缓存陈旧时，用户可一键清除本地缓存并刷新页面，同时保留 API Token 避免重新鉴权。

**范围边界**:
- 包含：清除 localStorage 缓存（保留 Token）、页面刷新、用户确认交互
- 不包含：服务端缓存清理、IndexedDB 清理、Token 失效处理

**依赖**: `STORAGE_KEYS.API_TOKEN`（C: YiWeb 项目需定义此常量）

## §1.1 User Operations

| 步骤 | 用户操作 | 系统响应 | 视图状态 |
|------|---------|---------|---------|
| 1 | 点击工具栏「清缓存并刷新」按钮 | 弹出 confirm 确认框 | 对话框阻断 |
| 2 | 点击「确定」 | 清除 localStorage（保留 Token），页面 reload | 页面白屏刷新 |
| 2a | 点击「取消」 | 无操作，关闭对话框 | 保持当前视图 |

```mermaid
    CLEAR --> RELOAD["location.reload()"]
    CONFIRM -->|取消| NOOP["无操作"]
```

## §2 Requirements

### 功能点

1. **确认对话框**（C: `src/views/aicr/index.js` 或全局入口）
   - 文案：`"确定要清空缓存并刷新页面？Token 将被保留。"`
   - 用户取消则中断流程

2. **缓存清除**（C: YiWeb 应用入口或 AICR 视图）
   - 遍历全部 localStorage 键
   - 保留 `STORAGE_KEYS.API_TOKEN`
   - 其余键全部 `localStorage.removeItem()`
   - 单键移除失败静默忽略（try/catch）

3. **页面刷新**（C: YiWeb 应用入口）
   - 调用 `location.reload()` 全页刷新

### 输入/输出

| 输入 | 类型 | 说明 |
|------|------|------|
| 用户点击 | Event | 按钮 `data-action="clearCacheAndRefresh"` |
| localStorage | Browser API | 当前域名下的全部键值对 |

| 输出 | 类型 | 说明 |
|------|------|------|
| 确认对话框 | boolean | `confirm()` 返回值 |
| localStorage 变更 | side effect | 非 Token 键被移除 |
| 页面刷新 | navigation | `location.reload()` |

### 业务规则

- 必须保留 API Token，避免用户重新输入鉴权信息
- 单键移除异常不得阻断后续键的清理
- 页面刷新为硬刷新，不保留表单状态

### 数据约束

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| preserveKey | string | 常量 | `STORAGE_KEYS.API_TOKEN`，不可变 |
| keysToRemove | string[] | 动态 | 运行时遍历 localStorage 得出 |

## §3 Design

### 技术方案

```
按钮触发层 (AICR 页面或全局)
  ↓ action === "clearCacheAndRefresh"
clearCacheAndRefresh()
  ├─ confirm() 用户确认
  ├─ 遍历 localStorage.key(i)
  ├─ 过滤 preserveKey
  ├─ removeItem(k) 批量清理
  └─ location.reload()
```

**核心逻辑**（C: YiWeb 实现位置待定）:
- 纯前端实现，无后端交互
- 依赖全局 `STORAGE_KEYS` 常量
- 使用原生 ES Modules 导出函数

### 安全约束

- 仅操作当前域名 localStorage，受同源策略保护
- Token 保留逻辑硬编码为 `STORAGE_KEYS.API_TOKEN`，避免误删
- 移除异常静默处理，防止异常键阻断流程

### 影响链

- **正向影响**: 用户可自助解决缓存导致的数据异常
- **反向影响**: 清除所有业务缓存，需重新加载
- **无服务端影响**: 纯客户端操作
