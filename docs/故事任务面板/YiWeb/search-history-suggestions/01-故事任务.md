## 需求来源

源码扫描发现 `cdn/utils/browser/events.js` 存在 3 处 TODO 标记：
- `handleArrowKeys`（L448）：`// TODO: 实现搜索历史导航`
- `showSearchHistory`（L512）：`// TODO: 实现搜索历史显示`
- `hideSearchHistory`（L519）：`// TODO: 实现搜索历史隐藏`

`SearchHandler` 类已具备历史记录的增删、持久化（localStorage `searchHistory`，max 10）、加载逻辑，但缺失 UI 渲染与键盘导航。

## 目标

为 YiWeb 全局搜索框补充「搜索历史下拉面板」与「搜索建议」功能，提升用户检索效率。

## 范围

### In Scope

1. **搜索历史下拉面板**
   - 聚焦搜索框时展示最近搜索历史（最多 10 条）
   - 支持鼠标点击历史项回填并触发搜索
   - 支持 `ArrowUp` / `ArrowDown` 键盘导航历史项
   - 支持 `Enter` 选中当前高亮项
   - 支持 `Escape` 关闭面板
   - 每条历史项右侧显示「删除」按钮（仅删除该条，不影响其他）
   - 面板底部显示「清空全部」按钮

2. **搜索建议（客户端过滤）**
   - 输入时从历史记录中过滤匹配项并展示
   - 匹配高亮（关键词加粗）
   - 无匹配时展示空提示「无匹配历史记录」

3. **无障碍**
   - 下拉面板 `role="listbox"`，历史项 `role="option"`
   - `aria-expanded` 绑定面板显隐
   - `aria-activedescendant` 跟踪当前高亮项
   - 键盘导航与屏幕阅读器兼容

### Out of Scope

- 服务端搜索建议 / 自动补全 API（当前无服务端支持）
- 跨设备搜索历史同步
- 搜索历史分类/标签

## 验收标准

| # | 标准 |
|---|------|
| 1 | 聚焦搜索框时，若历史非空，下拉面板在 100ms 内出现 |
| 2 | 输入字符时，面板从历史过滤并高亮匹配，无匹配展示空态 |
| 3 | `ArrowUp`/`ArrowDown` 可循环遍历历史/建议项，`Enter` 选中执行搜索 |
| 4 | 点击单项删除按钮仅移除该条；点击「清空全部」移除全部 |
| 5 | 面板失焦或 `Escape` 可关闭，再次聚焦恢复显示 |
| 6 | 所有历史操作实时持久化到 localStorage |
| 7 | 通过 axe-core 无障碍扫描（无 critical/serious 错误） |
