# 01-故事任务 — YiWeb-filetree-sidebar-ux

## 需求来源

基于 Story 1/2/3 建立的设计令牌，对文件树和侧边栏组件进行视觉升级，符合 UI UX Pro Max 的 SaaS/AI 平台设计规范。

## 需求摘要

使用新的 `--yi-*` 设计令牌，重写 `src/views/aicr/components/fileTree/` 和 `sessionListTags/` 下的组件样式：
- 统一为 Soft UI + Minimalism 风格
- 降低视觉噪音（减少重渐变、强发光、脉冲动画）
- 提升可访问性（焦点环、触摸目标、减少动画）
- 标准化过渡时长与缓动函数

| 维度 | 影响范围 | 证据等级 |

## 依赖

- **上游**: Story 1/2/3（设计令牌已就绪）
- **下游**: Story 5 `codeview-markdown-enhance`

## 优先级

P1

## 验收标准

1. 工具栏按钮取消线性渐变，改用纯色 + 微阴影
2. 文件树选中/高亮状态使用 `--yi-*` 令牌，无硬编码 indigo
3. 移除 `highlightFilePulse` / `warningPulse` 等脉冲动画（保留关键帧但简化）
4. 所有 `transition` 使用 `--yi-duration-*` 和 `--yi-easing-default`
5. `prefers-reduced-motion` 覆盖所有动画
