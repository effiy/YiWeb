# 01-故事任务 — YiWeb-codeview-markdown-enhance

## 需求来源

基于 Story 1/2/3/4 建立的设计令牌，对代码视图和 Markdown 渲染组件进行视觉升级，符合 UI UX Pro Max 的 SaaS/AI 平台设计规范。

## 需求摘要

使用 `--yi-*` 设计令牌，优化代码视图与 Markdown 渲染体验：
- 代码视图（codeView）组件样式令牌化
- 语法高亮样式引入语义令牌体系
- 清理 codePage.css 中残留的硬编码值与重复定义
- 降低视觉噪音（减少硬编码阴影、离散过渡）

## 影响分析

| 维度 | 影响范围 | 证据等级 |
|------|---------|---------|
| codeView | `src/views/aicr/components/codeView/index.css` | A |
| codePage | `src/views/aicr/styles/codePage.css` | A |
| syntax-highlight | `cdn/styles/components/syntax-highlight.css` | A |

## 依赖

- **上游**: Story 1/2/3/4（设计令牌已就绪）
- **下游**: Story 6 `micro-interactions`

## 优先级

P1

## 验收标准

1. codeView 图片预览过渡与阴影使用 `--yi-*` 令牌
2. codePage.css 无 fallback 硬编码色值（`#e9ecef` / `#f8f9fa`）
3. codePage.css 无离散 `transition` 时长（统一 `--yi-duration-*`）
4. syntax-highlight.css 引入语义语法高亮令牌（`--sh-keyword` / `--sh-string` 等）
5. `prefers-reduced-motion` 覆盖 codeView 动画
