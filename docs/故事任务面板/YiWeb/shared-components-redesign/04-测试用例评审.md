# 04-测试用例评审 — YiWeb-shared-components-redesign

## 测试策略

人工验证 + DevTools 检查清单。

## 测试用例

### TC-01: 按钮样式验证

| 项 | 内容 |
|----|------|
| 目的 | 确认 YiButton 各变体使用新令牌 |
| 步骤 | 1. 在浏览器打开 aicr 视图<br>2. 检查 Primary / Secondary / Outline / Ghost 按钮<br>3. DevTools Computed 确认无硬编码色值 |
| 预期 | 所有颜色来自 `--yi-*`，无渐变发光残留 |

### TC-02: 模态框动画

| 项 | 内容 |
|----|------|
| 目的 | 模态框动画符合 `--yi-duration-*` |
| 步骤 | 1. 触发任意模态框（如设置）<br>2. DevTools Animation 面板检查时长 |
| 预期 | 进入动画 ≈ 200ms，使用 `--yi-easing-default` |

### TC-03: 触摸目标

| 项 | 内容 |
|----|------|
| 目的 | 交互元素最小 44×44px |
| 步骤 | 1. DevTools → Elements → 检查按钮/标签/选项<br>2. 确认 width/height ≥ 44px |
| 预期 | 所有可点击元素满足 44×44px |

### TC-04: 焦点状态

| 项 | 内容 |
|----|------|
| 目的 | 焦点环可见且使用 `--yi-shadow-focus` |
| 步骤 | 1. Tab 键遍历页面<br>2. 观察按钮、输入框、选项的焦点环 |
| 预期 | 可见蓝色环（`#2563EB` 15% 透明度），无 outline:none 吞掉焦点 |

### TC-05: Reduced Motion

| 项 | 内容 |
|----|------|
| 目的 | `prefers-reduced-motion` 生效 |
| 步骤 | 1. OS 开启减少动画<br>2. 刷新页面，检查加载器、模态框、按钮 |
| 预期 | 加载器停止旋转，模态框无动画，按钮无过渡 |

### TC-06: 新增组件 YiErrorState

| 项 | 内容 |
|----|------|
| 目的 | 错误状态组件样式完整 |
| 步骤 | 1. 手动在 Console 触发错误场景或临时插入组件<br>2. 检查图标、文字、按钮布局 |
| 预期 | 红色主题，结构清晰，操作按钮可点击 |

### TC-07: 新增组件 YiSelect

| 项 | 内容 |
|----|------|
| 目的 | 下拉选择器样式完整 |
| 步骤 | 1. 点击模型选择器等下拉组件<br>2. 检查下拉面板、选项 hover、选中状态 |
| 预期 | 面板有阴影，选项 hover 背景变化，选中项高亮 |

## 回归范围

- `cdn/components/common/*/*/index.css`
- `src/views/aicr/index.html`（如组件加载顺序变更）

## 门禁标准

- TC-01 至 TC-05 为 P0，必须全部通过
- TC-06、TC-07 为 P1，允许一次修复
