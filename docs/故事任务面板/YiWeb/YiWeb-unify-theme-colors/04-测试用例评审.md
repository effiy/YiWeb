# 04 — 测试用例评审：统一主题色与基础组件

## 测试策略

零构建前端项目，无自动化测试框架。采用人工浏览器验证 + 静态分析扫描 + 结构化检查清单。

## Phase 1 测试用例：主题色统一

### TC-01：主题根文件完整性

| 项 | 内容 |
|---|---|
| 目的 | 确认 theme.css 规范层完整，遗留层已标记 |
| 步骤 | 1. 打开 `cdn/styles/theme.css` <br> 2. 检查 `:root` 内所有 `yi-*` token <br> 3. 检查遗留映射段是否有 `@deprecated` 注释 |
| 期望 | `yi-*` 覆盖 Primary/Semantic/Surfaces/Borders/Text/Shadows/Radius；遗留段有明确弃用标记；默认暗色值正确 |
| 通过标准 | 无缺失 token，标记清晰，默认值为暗色 |

### TC-02：CSS 遗留变量清零

| 项 | 内容 |
|---|---|
| 目的 | 确认 `src/` 下 CSS 无遗留变量引用 |
| 步骤 | `grep -r "\-\-primary\b\|\-\-bg-primary\b\|\-\-text-primary\b\|\-\-border-primary\b" src/ --include="*.css"` |
| 期望 | 结果为空 |
| 通过标准 | 0 处匹配 |

### TC-03：CSS 硬编码颜色清零

| 项 | 内容 |
|---|---|
| 目的 | 确认 `src/` 下 CSS 无业务硬编码颜色（文件类型品牌色除外） |
| 步骤 | `grep -rn "#[0-9a-fA-F]\{3,6\}" src/ --include="*.css"` |
| 期望 | 仅保留 `fileTreeTreeBase.css` 中 `.file-icon.*` 的品牌色（JS/TS/Vue 等 logo 色） |
| 通过标准 | 非文件图标类的硬编码为 0 |

### TC-04：JS 动态样式收敛

| 项 | 内容 |
|---|---|
| 目的 | 确认 JS 中无硬编码颜色字符串 |
| 步骤 | `grep -rn "#[0-9a-fA-F]\{3,6\}\|rgba(59,130,246\|rgba(0,0,0" src/ --include="*.js"` |
| 期望 | 无业务色硬编码 |
| 通过标准 | 0 处业务色匹配 |

### TC-05：暗色模式视觉回归

| 项 | 内容 |
|---|---|
| 目的 | 确认 `prefers-color-scheme: dark` 下无异常 |
| 步骤 | 1. 浏览器 DevTools → Rendering → Emulate `prefers-color-scheme: dark` <br> 2. 遍历：侧边栏、代码区、聊天面板、弹窗、搜索面板、快捷键帮助 |
| 期望 | 所有区域可读性正常，对比度无异常下降，无未覆盖的亮色块 |
| 通过标准 | 目视无异常 |

### TC-06：高对比度模式

| 项 | 内容 |
|---|---|
| 目的 | 确认 `prefers-contrast: high` 下可读性 |
| 步骤 | 浏览器 DevTools → Rendering → Emulate `prefers-contrast: high` |
| 期望 | 文本/背景对比度提升，边框可见性增强 |
| 通过标准 | 目视无异常 |

### TC-07：减少动效模式

| 项 | 内容 |
|---|---|
| 目的 | 确认 `prefers-reduced-motion: reduce` 下无动画残留 |
| 步骤 | 浏览器 DevTools → Rendering → Emulate `prefers-reduced-motion: reduce` |
| 期望 | 过渡动画时间为 0ms，无副作用 |
| 通过标准 | 过渡瞬时完成 |

### TC-08：主题色一致性

| 项 | 内容 |
|---|---|
| 目的 | 确认所有主色引用收敛到同一值 |
| 步骤 | 检查 `--yi-primary`、`--pet-chat-main-color`、resizer 拖拽条颜色是否一致 |
| 期望 | 三者均为 `#2563EB` 或其 rgba 变体 |
| 通过标准 | 色值一致 |

### TC-09：token 新增 RGB 分量可用性

| 项 | 内容 |
|---|---|
| 目的 | 确认新增的 `-rgb` token 可被 `rgba()` 使用 |
| 步骤 | 在浏览器控制台执行 `getComputedStyle(document.documentElement).getPropertyValue('--yi-dark-surface-rgb')` |
| 期望 | 返回有效 RGB 数值（如 `15, 23, 42`） |
| 通过标准 | 返回值非空且格式正确 |

### TC-10：默认暗色主题

| 项 | 内容 |
|---|---|
| 目的 | 确认 `:root` 默认值为暗色 |
| 步骤 | 检查 `:root` 中 `yi-bg`/`yi-surface`/`yi-text`/`yi-border`/`yi-shadow-*` 默认值 |
| 期望 | 背景/表面/文本/边框/阴影均为暗色优先值 |
| 通过标准 | 默认值匹配暗色规范 |

### TC-11：亮色模式覆盖

| 项 | 内容 |
|---|---|
| 目的 | 确认 `@media (prefers-color-scheme: light)` 恢复全部亮色值 |
| 步骤 | 检查 Light 媒体查询中 surface/border/text/subtle/shadow 共 17 个 token |
| 期望 | 全部恢复亮色值 |
| 通过标准 | 17 个 token 均已覆盖 |

### TC-12：对比度验证

| 项 | 内容 |
|---|---|
| 目的 | 关键文本/背景组合 WCAG 对比度 >= 4.5 |
| 步骤 | 使用对比度计算工具验证 `yi-text` on `yi-bg`、`yi-text-secondary` on `yi-surface` 等 |
| 期望 | 所有组合 >= 4.5 |
| 通过标准 | 计算值通过 |

## 回归检查清单

- [ ] Phase 1 已修复的 legacy 变量无回退
- [ ] YiButton 原有 variant（primary/secondary/outline/ghost/danger）不受影响
- [ ] YiTag 原有 variant（primary/success/warning/danger/info）不受影响
- [ ] YiSelect 原有交互（展开/选择/关闭）不受影响
- [ ] 视图布局无错位（尤其按钮+输入框并排场景）

## 已知不可测试项

- 文件类型图标颜色（`#f7df1e` 等）为第三方品牌标识色，不纳入主题系统，不测试收敛
- 浏览器实际渲染验证（零构建项目可直接打开验证，建议合并前执行）
