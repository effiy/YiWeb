### 需求输入

用户通过 `/rui` 提交需求：统一主题色与基础组件，清理源码中的遗留变量和硬编码颜色。

### 执行过程

1. **预检**：创建分支 `feat/YiWeb-unify-theme-colors`
2. **文档阶段**：生成 01/02/04/05 设计文档
3. **实现阶段（Phase 1：主题色统一）**：
   - 替换 123 处遗留 CSS 变量（`--primary`, `--bg-primary`, `--text-primary` 等）
   - 替换 28 处硬编码颜色值
   - 收敛 `--pet-chat-main-color` 至 `--yi-primary`
   - theme.css 遗留变量映射段标记 `@deprecated`
   - `:root` 默认改为暗色主题，亮色移至 `@media (prefers-color-scheme: light)`
4. **实现阶段（Phase 2：基础组件统一）**：
   - 新建 YiInput、YiTextarea 组件
   - 优化 YiButton（accent variant、block、lg 尺寸）
   - 优化 YiIconButton（size/variant 属性、基线 44px）
   - 优化 YiTag（accent variant、尺寸命名统一 sm/md/lg）
   - 优化 YiSelect（size 属性、CSS 类名修复）
   - 视图层 focus 状态统一收敛至 `var(--yi-border-focus)` + `var(--yi-shadow-focus)`
5. **验证阶段**：源码静态检查 + 结构化检查清单
6. **交付阶段**：生成 00/07/08/09 文档，执行 import-docs + wework-bot

### 关键决策

- 将 `:root` 默认设为暗色主题（项目实际运行模式），亮色变为可选媒体查询覆盖，彻底统一设计语义
- YiTag 尺寸保持独立 scale（24/28/36），不与控件族强行统一
- 视图内联输入本次仅收敛 focus Token，完整组件化替换留作后续 story

### 阻塞与解决

- `--yi-text-secondary` 无暗色覆盖，替换后暗色背景可读性降低 → 已在 dark media query 中补充覆盖
- YiSelect CSS `.select-trigger` 与模板实际类名不匹配 → 修复为 `.select`
- 无其他阻塞
