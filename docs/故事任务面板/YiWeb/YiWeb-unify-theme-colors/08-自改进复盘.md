## 关键发现

### 发现 1：暗色/亮色双轨设计

项目实际运行模式为**暗色主题**，但 `yi-*` token 按亮色优先设计。业务层存在两种使用模式：
- 暗色背景元素 → 使用遗留变量（映射到暗色值）
- 亮色表面元素 → 直接使用 `yi-*`（如 `--yi-surface` 按钮、欢迎卡片）

**影响**：迁移 `--text-secondary` 等文本变量时，若未补充暗色覆盖，会导致暗色背景上文本不可读。

**已修复**：将 `:root` 中通用 token 直接设为暗色值，`@media (prefers-color-scheme: light)` 中恢复亮色值。彻底消除了亮色/暗色双轨的割裂感。

### 发现 2：fallback 值分散

JS 内联样式字符串中大量使用 `var(--xxx, #value)` fallback，值分散且部分与主题不一致（如 `#6366f1`）。

**已修复**：统一 fallback 为当前主题色值。

### 发现 3：组件优先于内联样式

新建 YiInput/YiTextarea 后，后续视图输入替换有明确入口，避免继续扩散内联样式债务。

**已修复**：YiInput/YiTextarea 已新建并注册，视图 focus 状态已收敛。

### 发现 4：尺寸 scale 需分族群

按钮/输入/选择器共享控件 scale（36/44/52），标签因视觉层级不同保持独立 scale（24/28/36），强行统一反而破坏层级。

**已处理**：控件族统一 scale，标签保持独立 scale。

## 提案

### 提案 1：移除遗留变量映射段（P2）

当前 Legacy Map 已标记 `@deprecated`，但 `:root` 中仍保留大量遗留变量。建议彻底清理：

```css
/* 删除以下段 */
--primary: var(--yi-primary);
--bg-primary: var(--yi-dark-surface);
/* ... 约 180 行 */
```

**收益**：减少 theme.css 体积约 50%，消除新开发者误用风险。
**成本**：需全局搜索确认无引用（含 CDN 组件），约 1 小时。

### 提案 2：视图内联输入组件化替换（P2）

逐步将 `.aicr-session-settings-input`、`.aicr-session-faq-search-input`、`.pet-chat-textarea` 替换为 YiInput/YiTextarea 组件。

**收益**：消除内联样式债务，统一维护入口。
**成本**：约 2–3 小时，需验证表单交互。

### 提案 3：引入浏览器自动化快照测试（P2）

当前无浏览器测试框架。建议引入 Playwright 或 Puppeteer 做视觉快照：

```bash
# 伪代码
npm install -D playwright  # 但零构建链约束需评估
```

**收益**：TC-05~07 可自动化，防止主题色回归。
**成本**：需突破零构建约束或引入独立测试目录。

### 提案 4：清理遗留变量映射段（P3）

当前 Legacy Map 保留但标记 `@deprecated`。建议在 2–3 个迭代后彻底删除。

**收益**：减少 theme.css 体积（当前 360 行，Legacy Map 约 180 行）。
**成本**：需确认无外部 CDN 组件引用。
