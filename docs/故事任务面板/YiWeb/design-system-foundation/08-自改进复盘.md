## 改进项

1. **未来故事**：后续组件改造故事应逐步将组件内的 `var(--primary)` 迁移到 `var(--yi-primary)`，最终废弃旧变量名
2. **测试缺口**：建议引入 stylelint 做 CSS 变量引用检查
3. **暗色模式**：当前暗色模式令牌已预留，但切换逻辑尚未实现（可在 aicr-layout-ux 故事中补充）

## 经验沉淀

- 设计系统改造优先改 theme.css 和 utilities.css，不动组件源码，风险最低
- 字体选择对开发者工具类产品的感知质量影响显著
- 双命名空间策略适合存量项目渐进式升级
