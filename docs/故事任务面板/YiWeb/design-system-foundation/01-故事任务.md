# 01-故事任务 — YiWeb-design-system-foundation

## 需求来源

使用 [UI UX Pro Max](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill) 设计智能技能，对 YiWeb 全站进行 UI/UX 优化。本故事为系列第一环，负责建立设计系统基线。

## 需求摘要

为 YiWeb（AI 代码审查工具）建立一套现代、专业、可扩展的设计系统，覆盖色彩、字体、间距、阴影、动画令牌，并落地到 `cdn/styles/theme.css` 与全局工具类中。

## 影响分析

| 维度 | 影响范围 | 证据等级 |
|------|---------|---------|
| 样式变量 | `cdn/styles/theme.css` | A（直接修改） |
| 全局工具类 | `cdn/styles/utilities.css`, `cdn/styles/utils.css` | A（直接修改） |
| 基础样式 | `cdn/styles/base/*.css` | A（直接修改） |
| 共享组件 | `cdn/components/*` | B（消费令牌） |
| 视图组件 | `src/views/aicr/components/*` | B（消费令牌） |

## 故事拆分

本故事独立，不依赖其他故事。后续故事（shared-components-redesign、aicr-layout-ux 等）依赖本故事产出的设计令牌。

## 优先级

P0 — 所有后续 UI/UX 优化故事的前置依赖。

## 验收标准

1. `theme.css` 包含完整的 CSS 变量系统（颜色、字体、间距、阴影、圆角、动画）
2. 所有变量遵循 UI UX Pro Max 推荐的 SaaS/AI 平台设计规范
3. 现有视图在不修改组件代码的情况下，外观有可见提升（通过变量继承）
4. 无障碍：正文对比度 ≥ 4.5:1，大文本 ≥ 3:1
5. 暗色模式令牌已预留（`--yi-*-dark`）
