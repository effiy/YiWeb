# 04-测试用例评审 — YiWeb-design-system-foundation

## 测试策略

无自动化测试框架；采用人工验证 + 浏览器 DevTools 检查清单。

## 测试用例

### TC-01: 主题变量加载

| 项 | 内容 |
|----|------|
| 目的 | 确认 theme.css 变量在浏览器中正确解析 |
| 步骤 | 1. 打开 `src/views/aicr/index.html`<br>2. 打开 DevTools → Elements → 任意元素 → Computed → 筛选 `--yi-` |
| 预期 | 所有 `--yi-*` 变量有值，无 `invalid property value` |

### TC-02: 对比度合规

| 项 | 内容 |
|----|------|
| 目的 | 验证 WCAG AA 对比度 |
| 步骤 | 1. 使用 DevTools 的 Contrast checker<br>2. 检查 `--yi-text` on `--yi-surface`<br>3. 检查 `--yi-text-secondary` on `--yi-bg`<br>4. 检查 `--yi-text-on-primary` on `--yi-primary` |
| 预期 | 正文 ≥ 4.5:1，大文本 ≥ 3:1 |

### TC-03: 字体加载

| 项 | 内容 |
|----|------|
| 目的 | Inter 和 JetBrains Mono 正确加载 |
| 步骤 | 1. Network 面板筛选 `fonts.googleapis`<br>2. 检查字体是否 200<br>3. Elements 面板检查 `font-family` 是否生效 |
| 预期 | 无 404，字体栈正确回退到系统字体 |

### TC-04: 现有组件外观

| 项 | 内容 |
|----|------|
| 目的 | 变量变更不破坏现有组件布局 |
| 步骤 | 1. 打开 aicr 视图<br>2. 检查按钮、模态框、标签、文件树、代码区<br>3. 确认无错位、溢出、截断 |
| 预期 | 布局保持，颜色/字体按新变量更新 |

### TC-05: 工具类可用性

| 项 | 内容 |
|----|------|
| 目的 | utilities.css 中的工具类可正常使用 |
| 步骤 | 1. 在 DevTools Console 手动给元素添加 `class="yi-bg-surface yi-shadow-md yi-radius-lg"`<br>2. 观察样式变化 |
| 预期 | 背景、阴影、圆角正确生效 |

## 回归范围

- `src/views/aicr/index.html`
- `src/views/aicr/styles/index.css`
- `cdn/styles/theme.css`
- `cdn/styles/utilities.css`

## 门禁标准

- 所有 TC 通过后方可进入实现阶段
- 任一 TC 失败 → 修复后重验
