# 01-故事任务 — YiWeb-aicr-layout-ux

## 需求来源

基于 Story 1 `design-system-foundation` 建立的设计令牌，对 aicr 主视图布局样式进行视觉升级，符合 UI UX Pro Max 的 SaaS/AI 平台设计规范。

## 需求摘要

使用新的 `--yi-*` 设计令牌，重写 `src/views/aicr/styles/` 下的布局与视图样式：
- 统一为 Soft UI + Minimalism 风格
- 降低视觉噪音（减少重渐变、强发光、霓虹色边缘引导）
- 提升可访问性（焦点环、对比度、触摸目标、减少动画）
- 标准化过渡时长与缓动函数

## 影响分析

| 维度 | 影响范围 | 证据等级 |
|------|---------|---------|
| Layout | `src/views/aicr/styles/layout.css` | A |
| Code Page | `src/views/aicr/styles/codePage.css` | A |
| Context Modals | `src/views/aicr/styles/codePage.contextModals.css` | A |
| Welcome Card | `src/views/aicr/styles/welcomeCard.css` | A |

## 依赖

- **上游**: Story 1 `design-system-foundation`（设计令牌必须已就绪）
- **上游**: Story 2 `shared-components-redesign`（共享组件样式必须已就绪）
- **下游**: Story 4 `filetree-sidebar-ux`

## 优先级

P1 — 依赖 Story 1/2，必须在视图级组件优化之前完成。

## 验收标准

1. 布局文件使用 `--yi-*` 令牌替代硬编码 rgba 值
2. 侧边栏展开把手、文件树切换按钮取消霓虹发光与脉冲动画
3. 欢迎卡片、空状态卡片简化渐变，使用 surface + border 风格
4. 所有 `transition` 使用 `--yi-duration-*` 和 `--yi-easing-default`
5. `prefers-reduced-motion` 覆盖所有动画（含脉冲、淡入、滑入）
6. 聊天输入框、气泡、模态框使用语义颜色令牌
