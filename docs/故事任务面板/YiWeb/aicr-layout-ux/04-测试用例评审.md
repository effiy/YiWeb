# 04-测试用例评审 — YiWeb-aicr-layout-ux

## 测试策略

人工验证 + DevTools 检查清单。

## 测试用例

### TC-01: 布局令牌覆盖

| 项 | 内容 |
|----|------|
| 目的 | 确认 layout.css 使用 `--yi-*` 令牌 |
| 步骤 | 1. DevTools Computed 检查侧边栏、把手、按钮背景<br>2. 确认无硬编码 indigo rgba 残留 |
| 预期 | 颜色来自 `--yi-*`，无 `rgba(79,70,229,...)` 残留 |

### TC-02: 动画标准化

| 项 | 内容 |
|----|------|
| 目的 | 过渡使用 `--yi-duration-*` |
| 步骤 | 1. DevTools Animation 面板检查侧边栏展开、模态框进入<br>2. 确认时长 ≈ 200ms |
| 预期 | 所有过渡使用令牌值，无离散 `0.3s` 硬编码 |

### TC-03: Reduced Motion

| 项 | 内容 |
|----|------|
| 目的 | `prefers-reduced-motion` 生效 |
| 步骤 | 1. OS 开启减少动画<br>2. 检查侧边栏、把手、空状态卡片 |
| 预期 | 脉冲动画停止，滑入变为直接显示 |

### TC-04: 空状态卡片

| 项 | 内容 |
|----|------|
| 目的 | 空状态卡片使用 Soft UI |
| 步骤 | 1. 打开无代码文件状态<br>2. 观察空状态卡片样式 |
| 预期 | 无渐变发光，使用 surface + border + shadow |

### TC-05: 聊天气泡

| 项 | 内容 |
|----|------|
| 目的 | 聊天气泡使用语义令牌 |
| 步骤 | 1. 发送消息触发用户气泡<br>2. 观察 AI 回复气泡 |
| 预期 | 用户气泡 `--yi-primary`，AI 气泡 `--yi-surface` |

### TC-06: 模态框聚焦

| 项 | 内容 |
|----|------|
| 目的 | 输入框聚焦环使用 `--yi-shadow-focus` |
| 步骤 | 1. 打开上下文编辑模态框<br>2. Tab 到输入框 |
| 预期 | 可见蓝色环，无青色 `rgba(34,211,238,...)` 残留 |

## 回归范围

- `src/views/aicr/styles/*.css`
- `src/views/aicr/index.html`

## 门禁标准

- TC-01 至 TC-04 为 P0，必须全部通过
- TC-05、TC-06 为 P1，允许一次修复
