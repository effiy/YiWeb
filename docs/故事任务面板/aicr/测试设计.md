### 正常路径

#### TC-S1-N01: 文件树加载
- **Given**: 后端 API 返回会话数据
- **When**: 页面加载
- **Then**: 文件树从 session 数据构建并展示，统计栏显示文件/会话数量

#### TC-S1-N02: 代码文件查看
- **Given**: 文件树已加载，存在 .js 文件
- **When**: 点击 .js 文件
- **Then**: 代码区展示语法高亮代码，行号可点击复制链接

#### TC-S1-N03: Markdown 查看
- **Given**: 文件树已加载，存在 .md 文件
- **When**: 点击 .md 文件
- **Then**: 代码区渲染 Markdown 内容，支持内部链接跳转

#### TC-S1-N04: 图片预览
- **Given**: 文件树已加载，存在 .png 文件
- **When**: 点击 .png 文件
- **Then**: 弹出图片全屏预览弹窗

#### TC-S1-N05: URL 定位
- **Given**: URL 含 `?key=<path>` 参数
- **When**: 页面加载
- **Then**: 自动展开对应目录层级，定位并高亮目标文件，加载其内容

#### TC-S1-N06: 视图切换
- **Given**: 文件树已加载为树形视图
- **When**: 点击视图切换按钮
- **Then**: 切换为卡片视图，文件以卡片形式展示；再次点击切回树形

#### TC-S1-N07: 文件名搜索
- **Given**: 文件树已加载
- **When**: 输入搜索关键词
- **Then**: 300ms 防抖后过滤结果显示匹配文件，不匹配的文件隐藏

#### TC-S1-N08: 锚点跳转滚动
- **Given**: 已打开一份含目录锚点链接的 Markdown 文档
- **When**: 点击目录中的锚点链接（如 `[§1 场景 1](#s-1-场景-1-代码审查者浏览文件)`）
- **Then**: 页面平滑滚动到对应标题位置，目标标题可见且在视口内

### 边界/异常

#### TC-S1-E01: API 不可达
- **Given**: 网络断开或 API 服务宕机
- **When**: 页面加载
- **Then**: 错误状态展示 + 重试按钮，文件树显示空状态

#### TC-S1-E02: 文件加载失败
- **Given**: 文件树正常展示
- **When**: 点击文件后 API 返回 500
- **Then**: 代码区显示错误占位提示，不影响左侧文件树状态

#### TC-S1-E03: 搜索无匹配
- **Given**: 文件树已加载
- **When**: 搜索一个不存在的文件名
- **Then**: 空状态提示"未找到匹配文件"，提供清除搜索按钮

#### TC-S1-E04: 锚点目标不存在
- **Given**: 已打开一份 Markdown 文档
- **When**: 点击一个指向不存在标题的锚点链接（href 的 id 在文档中不存在）
- **Then**: 页面不发生跳转，不报错，不出现空白区域

### 正常路径

#### TC-S2-N01: 流式聊天
- **Given**: 已选中文件作为上下文，Ollama 服务可用
- **When**: 输入"解释这段代码"并发送
- **Then**: SSE 流式响应逐块展示在聊天面板，文件内容作为上下文随请求发送

#### TC-S2-N02: 重新生成
- **Given**: 上一条 AI 回复已展示
- **When**: 点击重新生成按钮
- **Then**: 清除上一条回复内容，使用相同上下文重新请求，新回复流式展示

#### TC-S2-N03: 复制回复
- **Given**: AI 回复已完整展示
- **When**: 点击复制按钮
- **Then**: 回复内容复制到剪贴板，显示复制成功提示

#### TC-S2-N04: 编辑上下文
- **Given**: 已选中文件
- **When**: 打开上下文编辑器，编辑文件内容，保存
- **Then**: 下次聊天使用编辑后的上下文发送请求

### 边界/异常

#### TC-S2-E01: Ollama 不可用
- **Given**: Ollama 服务宕机
- **When**: 发送聊天消息
- **Then**: 错误提示"模型服务不可用"，不展示空白回复

#### TC-S2-E02: SSE 中断
- **Given**: 流式聊天进行中
- **When**: 网络断开
- **Then**: "流式连接中断"提示，已接收的部分内容保留在聊天面板

#### TC-S2-E03: 空上下文
- **Given**: 未选择任何文件
- **When**: 输入消息并发送
- **Then**: 使用默认上下文正常发送，不报错

### 正常路径

#### TC-S3-N01: 项目标签
- **Given**: 文件树包含多个项目目录
- **When**: 点击"YiWeb"项目标签
- **Then**: 文件树仅显示 YiWeb 根目录内容，统计栏更新文件数量，故事标签选项联动更新

#### TC-S3-N02: 故事标签
- **Given**: 已选择"YiWeb"项目标签
- **When**: 点击"aicr"故事标签
- **Then**: 文件树进一步缩小为 aicr 子目录，状态标签选项联动更新

#### TC-S3-N03: 状态标签
- **Given**: 已选择项目标签和故事标签
- **When**: 点击"技术评审"状态标签
- **Then**: 仅显示匹配"技术评审"类型的文件，其他文件隐藏

#### TC-S3-N04: 无标签筛选
- **Given**: 某项目根目录存在无子目录的孤立文件
- **When**: 点击"无标签"筛选按钮
- **Then**: 仅显示根目录下无子目录的文件，有子目录的文件隐藏

#### TC-S3-N05: Escape 清除
- **Given**: 已应用任意筛选条件
- **When**: 按 Escape 键
- **Then**: 所有筛选条件清除，文件树恢复完整列表，统计栏恢复全量数字

#### TC-S3-N06: 按钮清除
- **Given**: 已应用任意筛选条件
- **When**: 点击清除筛选按钮
- **Then**: 同 Escape 清除，所有筛选条件重置

### 边界/异常

#### TC-S3-E01: 组合无结果
- **Given**: 已选择项目 + 故事 + 状态标签
- **When**: 该组合下无任何匹配文件
- **Then**: 显示空状态提示 + "清除筛选"快捷按钮，不报错

#### TC-S3-E02: 快速切换
- **Given**: 已选择"aicr"故事标签
- **When**: 快速切换选择"claude"故事标签
- **Then**: 文件树立即更新为 claude 目录内容，状态标签联动更新，无闪烁或中间态

    NORMAL -.->|"覆盖"| EDGE
```

### 正常路径

#### TC-S4-N01: 创建会话
- **Given**: 会话列表已加载
- **When**: 点击创建会话按钮
- **Then**: 新会话出现在列表中，自动获得默认名称，可立即开始编辑

#### TC-S4-N02: 编辑元数据
- **Given**: 会话列表中存在会话
- **When**: 编辑会话名称或描述，保存
- **Then**: 会话元数据更新，列表刷新显示新名称

#### TC-S4-N03: 标签管理
- **Given**: 会话已存在
- **When**: 为会话添加标签"review"，再移除该标签
- **Then**: 添加后标签出现在会话标签列表中，移除后标签消失

#### TC-S4-N04: 批量删除
- **Given**: 会话列表存在多个会话
- **When**: 进入批量模式，多选 3 个会话，点击删除并确认
- **Then**: 3 个会话从列表中移除，后端同步删除

#### TC-S4-N05: 收藏会话
- **Given**: 会话列表中存在会话
- **When**: 点击会话的收藏按钮
- **Then**: 会话标记为已收藏，收藏列表中出现该会话；再次点击取消收藏

### 边界/异常

#### TC-S4-E01: 删除失败
- **Given**: 会话列表正常
- **When**: 删除会话时 API 返回 500
- **Then**: 错误提示"删除失败"，会话仍保留在列表中不消失

#### TC-S4-E02: 部分失败
- **Given**: 已选中 3 个会话进入批量删除
- **When**: API 返回部分成功
- **Then**: 成功的从列表移除，失败的保留并提示具体失败项

### 正常路径

#### TC-S5-N01: 模型列表加载
- **Given**: Ollama 服务可用
- **When**: 打开模型选择器
- **Then**: 从 Ollama 拉取可用模型列表，以下拉选项展示

#### TC-S5-N02: 列表选择模型
- **Given**: 模型列表已加载
- **When**: 从下拉列表选择一个模型
- **Then**: 选中模型 ID 更新，后续聊天请求使用该模型

#### TC-S5-N03: 手动输入 ID
- **Given**: Ollama 服务不可达或列表为空
- **When**: 手动输入模型 ID 并确认
- **Then**: 使用手动输入的模型 ID 发送请求，不依赖模型列表

#### TC-S5-N04: 系统提示词
- **Given**: 用户打开设置面板
- **When**: 编辑系统提示词内容，保存
- **Then**: 保存成功，下次聊天请求自动注入该提示词

### 边界/异常

#### TC-S5-E01: 列表加载失败
- **Given**: Ollama 服务不可用
- **When**: 打开模型选择器或点击刷新
- **Then**: 显示"无法获取模型列表"提示，允许手动输入模型 ID 继续使用

#### TC-S5-E02: 提示词为空
- **Given**: 用户清空系统提示词
- **When**: 保存空提示词并发送聊天
- **Then**: 请求中不注入系统提示词，正常发送，不报错

    end

```

### 正常路径

#### TC-S6-N01: 创建文件
- **Given**: 右键点击文件夹打开上下文菜单
- **When**: 选择"新建文件"，输入文件名，确认
- **Then**: 新文件出现在树中对应位置，后端同步创建

#### TC-S6-N02: 创建文件夹
- **Given**: 右键点击父目录
- **When**: 选择"新建文件夹"，输入名称，确认
- **Then**: 新文件夹出现在树中，可继续在其中创建文件

#### TC-S6-N03: 重命名
- **Given**: 右键点击文件或文件夹
- **When**: 选择"重命名"，输入新名称，确认
- **Then**: 树节点名称更新，后端同步重命名

#### TC-S6-N04: 删除
- **Given**: 右键点击文件或文件夹
- **When**: 选择"删除"，在确认对话框中确认
- **Then**: 节点从文件树中移除，后端同步删除

#### TC-S6-N05: 导入文件夹
- **Given**: 点击工具栏"导入文件夹"
- **When**: 选择本地文件夹并确认
- **Then**: 文件夹及其内容上传到后端，文件树中显示新内容

#### TC-S6-N06: 导出 ZIP
- **Given**: 文件树已加载
- **When**: 点击"导出文件夹"选中目录或点击"项目 ZIP 下载"
- **Then**: 浏览器下载 ZIP 包，包含对应目录的全部文件

### 边界/异常

#### TC-S6-E01: 重命名冲突
- **Given**: 文件树中存在文件 A
- **When**: 将文件 B 重命名为 A 的名称
- **Then**: "文件名已存在"错误提示，重命名操作取消，文件 B 保持原名

#### TC-S6-E02: 导入失败
- **Given**: 选择了本地文件夹
- **When**: 上传过程中网络中断或后端返回错误
- **Then**: 错误提示"导入失败"，文件树保持不变

#### TC-S6-E03: ZIP 损坏
- **Given**: 上传项目 ZIP
- **When**: ZIP 文件格式损坏或内容不合法
- **Then**: "ZIP 文件无效"错误提示，不更新文件树
