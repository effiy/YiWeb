# 故事任务

> **基线类型**：问题空间 · **版本**：2.1.1 · **生成日期**：2026-05-27

## 概述

AICR（AI Code Review）是 YiWeb 的代码审查工作区。用户在一个面板中完成：浏览项目文件结构、查看文件内容、与 AI 对话分析代码、通过标签筛选快速定位文档。

面向的用户角色：代码审查者、项目维护者、新加入团队的成员。

### 主要价值

- 🔍 **一站式审查** — 文件浏览、内容查看、AI 分析在同一面板完成，无需切换工具
- 💬 **AI 辅助理解** — 查看代码时直接向 AI 提问，获得上下文相关的分析和建议
- 🏷️ **快速定位** — 通过项目→模块→文档类型三级标签逐级缩小范围，在大量文件中找到目标
- 📁 **灵活组织** — 按项目、模块、文档类型自由组织文件结构，支持导入导出
- 📝 **会话留存** — 每次 AI 对话保存为会话，可回顾、复用、整理
- 🚫 **明确的不做清单** — 不做 Git 集成、不做多人协作、不做离线模式，保持产品边界清晰

## §2 范围边界

### 范围内（做什么）

| 条目 | 关联 FP# | 产品决策依据 |
|------|---------|-------------|
| 文件树层级浏览（树形/卡片两种视图） | FP1, FP8 | 用户需要直观了解项目结构 |
| 文件内容渲染（代码高亮、Markdown 预览、图片预览） | FP2 | 审查者最常见的三种文件类型 |
| AI 流式对话（携带文件上下文） | FP3 | 审查者需要即时获得代码分析反馈 |
| 对话上下文编辑 | FP4 | 用户需要控制发送给 AI 的背景信息 |
| AI 模型切换 | FP5 | 不同场景可能需要不同模型 |
| 项目→模块→文档类型三级标签联动筛选 | FP6 | 用户在大量文件中快速定位的核心路径 |
| 文件名关键词模糊搜索 | FP7 | 标签筛选的补充，覆盖用户已知文件名的场景 |
| 会话 CRUD 与标签管理 | FP9, FP10, FP11 | 用户需要组织和回顾历史对话 |
| 文件/文件夹创建、重命名、删除 | FP12 | 项目维护的基本操作 |
| 文件夹导入导出、项目 ZIP 打包 | FP13, FP14 | 项目迁移和备份场景 |
| FAQ 查阅 | FP15 | 新用户上手和常见问题自助解决 |
| 面板布局拖拽调整 + 持久化记忆 | — | 用户个性化工作区 |
| 文件路径直达（URL 参数定位） | — | 外部链接和书签场景 |

### 范围外（不做什么）

| 条目 | 排除原因 | 产品决策 |
|------|---------|---------|
| **文件内容在线编辑** | 编辑器是独立产品范畴，复杂度远超审查面板定位 | AICR 定位为「审查」而非「编辑」，保持工具边界 |
| **Git 版本控制集成**（diff 对比、分支切换、commit 历史） | Git 操作有专门的工具和 UI 范式，不应内嵌到审查面板 | 审查者如需 Git 功能应使用专用 Git 工具 |
| **全文搜索**（搜索文件内容而非仅文件名） | 全文搜索需要索引构建和后端支持，非当前阶段优先级 | 文件名搜索覆盖主要定位场景；未来可作为独立 Story 评估 |
| **多人实时协作**（同时编辑、光标同步、评论批注） | 需要 WebSocket 基础设施和冲突解决机制，超出面板功能范畴 | AICR 是个人工作区，非协作平台 |
| **离线模式** | AICR 依赖后端 API 提供文件数据和 AI 服务 | 不投入资源支持离线场景 |
| **对话导出**（导出为 Markdown/PDF/分享链接） | 属于知识管理功能，非审查核心流程 | 用户可通过复制粘贴自行保存；未来可评估 |
| **自定义语法高亮主题** | 主题系统属于 IDE 级别功能 | 使用统一默认主题，保持体验一致 |
| **筛选条件保存为书签或预设** | 筛选条件组合过于个性化，保存/管理预设增加复杂度 | 筛选操作足够轻量，无需持久化筛选方案 |
| **会话版本对比**（对比同一会话的不同轮次） | 对话历史已按时间排列，对比非核心需求 | 用户可滚动查看历史消息 |
| **文件权限控制**（只读/可写/管理员） | AICR 面向单用户使用场景 | 不引入权限模型 |
| **正则表达式搜索** | 大多数用户不需要正则搜索能力 | 保持搜索简单直观 |
| **批量文件上传**（拖拽多文件上传） | 单文件导入已覆盖主要场景 | 批量场景使用 ZIP 打包导入 |

## §3 AC

| Given | When | Then |
|-------|------|------|
| 用户打开 AICR 面板 | 页面加载完成 | 文件树加载并展示，无数据时展示空状态引导 |
| 文件树已加载 | 用户点击文件节点 | 文件内容按类型正确渲染（代码/Markdown/图片） |
| 文件内容正在展示 | 用户输入消息并发送 | AI 流式回复逐字展示，内容关联当前文件 |
| 文件树包含多个项目 | 用户选择项目标签 | 文件树仅展示该项目内容，统计信息同步更新 |
| 已选择项目标签 | 用户选择模块标签 | 文件树进一步缩小范围，文档类型标签可选范围联动更新 |
| 已选择模块标签 | 用户选择文档类型标签 | 展示最终匹配的文件列表 |
| 文件树已展示 | 用户输入搜索关键词 | 输入停止后即时展示文件名匹配的结果 |
| 当前有筛选条件生效 | 用户点击清除或按 Escape | 所有筛选条件清除，恢复完整文件列表 |
| 会话列表已加载 | 用户创建/编辑/删除会话 | 操作执行成功，列表即时更新 |

| 用户调整面板布局（拖拽分隔条） | 用户刷新页面或下次打开 | 布局保持用户上次调整的结果 |

## §4 风险与假设

### 风险

- ⚠️ **文件数量过大导致性能下降** — 可能性：低 · 影响：中 · 缓解：监控文件规模；达到阈值时评估后端分页或虚拟滚动方案
- ⚠️ **AI 服务不可用导致聊天功能无法使用** — 可能性：中 · 影响：中 · 缓解：允许手动输入模型标识作为降级方案；展示清晰的服务状态提示
- ⚠️ **AI 回复中断导致对话不完整** — 可能性：中 · 影响：低 · 缓解：自动重连并展示中断提示；用户可重新发送消息
- ⚠️ **用户以为搜索是全文搜索而非文件名搜索** — 可能性：中 · 影响：低 · 缓解：搜索框占位文本明确提示"按文件名搜索"
- ⚠️ **文件树结构与用户心智模型不一致** — 可能性：低 · 影响：中 · 缓解：文件树结构由后端数据驱动，与项目实际结构保持一致

### 假设

- 💡 **用户期望在审查面板中直接编辑文件** — 范围外已明确排除；如用户反馈强烈可重新评估优先级
- 💡 **AI 模型列表可由服务端正常获取** — 如获取失败，允许手动输入模型标识作为降级方案
- 💡 **文件内容大小在可渲染范围内（非巨大文件）** — 超大文件展示截断提示
- 💡 **用户的主要使用场景是审查而非编辑** — 产品方向基于此假设；如数据表明编辑需求强烈，重新评估边界
