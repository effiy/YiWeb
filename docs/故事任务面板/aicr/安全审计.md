## §1 STRIDE 威胁建模

### S — Spoofing（身份伪造）

- 🎭 **伪造 X-Token 访问 API** — 缓解：Token 由后端签发和验证，前端仅透传
- 🎭 **伪造模型 ID 注入恶意模型** — 缓解：模型 ID 仅用于请求构造，后端实际执行
- 🎭 **伪造会话数据覆盖文件树** — 缓解：数据来源为认证 API，不接受客户端注入

### T — Tampering（数据篡改）

- ✏️ **聊天消息注入 XSS** — 缓解：SSE 内容经 textContent 渲染，非 innerHTML
- ✏️ **文件内容篡改** — 缓解：文件保存走 API 验证，非直接写文件系统
- ✏️ **标签排序篡改 localStorage** — 缓解：序列化为 JSON 存储在客户端，仅影响展示

### R — Repudiation（抵赖）

- 📋 **无操作日志** — 缓解：使用 logInfo/logError 记录关键操作到控制台
- 📋 **无法追溯修改者** — 缓解：后端 API 记录操作者和时间戳

### I — Information Disclosure（信息泄漏）

- 👁️ **API 响应泄露敏感文件内容** — 缓解：API 层验证权限，响应仅包含授权数据
- 👁️ **localStorage 存 X-Token 明文** — 缓解：Token 本身为 JWT，由后端控制过期
- 👁️ **调试日志输出敏感信息** — 缓解：生产环境 debug=false，logInfo 输出受限
- 👁️ **URL 参数泄露文件路径** — 缓解：`?key=<path>` 仅用于定位已授权文件

### D — Denial of Service（拒绝服务）

- 💥 **大量文件导致客户端 OOM** — 缓解：文件树懒加载 + computed 过滤，不渲染全部节点
- 💥 **流式响应洪水** — 缓解：SSE 逐块接收，浏览器原生背压
- 💥 **恶意 Zip 炸弹** — 缓解：Zip 上传后在服务端处理

### E — Elevation of Privilege（权限提升）

- 🔓 **通过文件树 CRUD 越权删除** — 缓解：后端 API 验证操作权限
- 🔓 **通过会话修改覆盖他人数据** — 缓解：会话 key 为唯一标识，API 层隔离
- 🔓 **修改 localStorage 切换环境** — 缓解：仅影响展示 URL，不绕过认证

## §2 输入面审计

- 🔍 **搜索框 input** — 类型：用户文本 · 验证：trim 后匹配，无 SQL/HTML 注入 · 风险：低
- 💬 **聊天消息 input** — 类型：用户文本 · 验证：发送前转义，SSE 响应用 textContent · 风险：中
- 📝 **会话上下文编辑器** — 类型：用户文本 · 验证：保存到 API，后端处理 · 风险：中
- ⚙️ **系统提示词** — 类型：配置文本 · 验证：仅作为 API 参数传递 · 风险：中
- 📦 **文件上传 (ZIP/文件夹)** — 类型：二进制 · 验证：后端处理，客户端仅传输 · 风险：中
- 🖼️ **粘贴图片 (聊天/代码编辑)** — 类型：base64 · 验证：上传 OSS 后获取 URL · 风险：中
- 🔗 **URL 参数 (key/tag/startLine/endLine)** — 类型：string · 验证：URLSearchParams 解析，验证范围 · 风险：低
- 📄 **Markdown 渲染** — 类型：HTML · 验证：SanitizePlugin 过滤，阻止 XSS · 风险：高（已缓解）
- 🤖 **Ollama 模型列表** — 类型：JSON · 验证：仅用于下拉列表展示 · 风险：低
- 🔑 **localStorage (X-Token)** — 类型：string · 验证：getStoredToken() 读取，验证 JWT 格式 · 风险：中
