## §1 使用场景

### 场景 1: 代码审查者浏览文件

**角色**: 代码审查者
**目标**: 浏览项目文件树并查看文件内容

```mermaid
flowchart LR
    A["页面加载"] --> B["构建文件树"]
    B --> C["浏览目录"]
    C --> D["展开文件夹"]
    D --> E["点击文件"]
    E --> F{"文件类型?"}
    F -->|"代码"| G["语法高亮展示"]
    F -->|"Markdown"| H["Markdown 渲染"]
    F -->|"图片"| I["图片预览"]
    G --> J["行号点击复制链接"]
    H --> K["Markdown 内链跳转"]
    I --> L["全屏预览"]
```

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 打开 AICR 面板 | 文件树在左侧加载 |
| 2 | 展开项目文件夹 | 显示子目录和文件 |
| 3 | 切换到卡片视图 | 文件以卡片形式展示 |
| 4 | 点击 `.js` 文件 | 代码区展示带语法高亮的代码 |
| 5 | 点击 `.md` 文件 | 代码区渲染 Markdown |
| 6 | 点击 `.png` 文件 | 图片预览弹窗 |

---

### 场景 2: 开发者 AI 代码分析

**角色**: 开发者
**目标**: 选中文件并让 AI 分析代码逻辑

```mermaid
flowchart LR
    A["点击文件"] --> B["文件加载到代码区"]
    B --> C["文件成为聊天上下文"]
    C --> D["输入问题"]
    D --> E["发送消息"]
    E --> F{"Ollama 可用?"}
    F -->|"是"| G["SSE 流式响应"]
    F -->|"否"| H["显示错误"]
    G --> I["查看 AI 回复"]
    I --> J["继续对话或切换文件"]
```

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 选中目标文件 | 文件内容展示在代码区 |
| 2 | 在聊天框输入"解释这段代码" | 构造带文件上下文的请求 |
| 3 | 发送消息 | SSE 流式展示 AI 回复 |
| 4 | 重新生成回复 | 清除上一条，重新请求 |
| 5 | 复制 AI 回复 | 内容复制到剪贴板 |

---

### 场景 3: 管理者四级联动筛选文件

**角色**: 文档管理者
**目标**: 通过标签系统快速定位特定类型的文档

```mermaid
flowchart LR
    A["全部文件展示"] --> B["选择'YiWeb'项目标签"]
    B --> C["仅显示 YiWeb 文件"]
    C --> D["选择'故事任务面板'故事标签"]
    D --> E["仅显示故事面板目录"]
    E --> F["选择'claude'前缀标签"]
    F --> G["仅显示 claude/ 前缀文件"]
    G --> H["选择'技术评审'后缀标签"]
    H --> I["精确显示 claude 目录的技术评审文档"]
    I --> J["点击文件查看"]
```

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 点击"YiWeb"项目标签 | 文件树缩减为 YiWeb 根目录 |
| 2 | 点击子目录"故事任务面板" | 仅显示该目录下的文件 |
| 3 | 点击"安全审计"后缀标签 | 仅显示安全审计类文件 |
| 4 | 点击"无标签"按钮 | 仅显示根目录下无目录的文件 |
| 5 | 按 Escape | 全部筛选清除，恢复完整列表 |

### 场景 5: 新成员接入 API 聊天

**角色**: 新成员
**目标**: 选择 AI 模型并配置系统提示词后开始使用

```mermaid
flowchart LR
    A["打开模型选择器"] --> B{"Ollama 可用?"}
    B -->|"是"| C["从下拉列表选择"]
    B -->|"否"| D["手动输入模型 ID"]
    C --> E["点击刷新重拉列表"]
    D --> E
    E --> F["打开系统提示词设置"]
    F --> G["编辑保存提示词"]
    G --> H["开始聊天"]
```
