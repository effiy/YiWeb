## §1 使用场景

### 场景 1: 审查者浏览和查找会话

**角色**: 代码审查者
**目标**: 快速找到需要审查的会话

```mermaid
flowchart LR
    A["打开页面"] --> B["会话列表加载"]
    B --> C{"列表为空?"}
    C -->|"是"| D["显示空状态提示"]
    C -->|"否"| E["浏览会话列表"]
    E --> F["使用搜索框过滤"]
    F --> G["点击标签筛选"]
    G --> H["找到目标会话"]
    H --> I["点击会话开始审查"]
```

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 打开代码审查页面 | 会话列表自动加载，显示加载中状态 |
| 2 | 在搜索框输入项目名 | 列表实时过滤，仅显示匹配的会话 |
| 3 | 点击一个标签 | 列表进一步缩小到含该标签的会话 |
| 4 | 点击目标会话 | 会话高亮选中，文件树和聊天面板加载 |

**空状态**: 无会话时显示引导提示
**错误恢复**: 加载失败显示重试按钮和错误原因

---

### 场景 2: 审查者浏览文件并查看代码

**角色**: 代码审查者
**目标**: 浏览项目文件结构，查看具体代码内容

```mermaid
flowchart LR
    A["会话已选中"] --> B["文件树加载"]
    B --> C["展开目录"]
    C --> D["浏览文件列表"]
    D --> E["点击文件名"]
    E --> F{"文件类型?"}
    F -->|"代码"| G["代码高亮显示"]
    F -->|"Markdown"| H["渲染预览"]
```

**空状态**: 会话无关联文件时显示"项目为空"
**错误恢复**: 文件内容加载失败时显示错误占位

---

### 场景 3: 审查者与 AI 讨论代码

**角色**: 代码审查者
**目标**: 在聊天面板中向 AI 提问，获得代码审查建议

```mermaid
flowchart LR
    A["输入问题"] --> B["点击发送"]
    B --> C["消息出现在聊天区"]
    C --> D["AI 流式响应"]
    D --> E["逐字渲染"]
    E --> F{"响应完成?"}
    F -->|"正常结束"| G["完整回复"]
    F -->|"中断"| H["保留已接收内容"]
    F -->|"错误"| I["显示错误提示"]
```

**空状态**: 新会话无历史消息，显示欢迎卡片
**错误恢复**: 发送失败时消息保留在输入框，可重新发送

---

### 场景 4: 审查者管理上下文和编辑消息

**角色**: 代码审查者
**目标**: 控制 AI 对话的上下文范围，编辑或撤回消息

```mermaid
flowchart LR
    A["对话进行中"] --> B{"操作?"}
    B -->|"编辑消息"| C["点击编辑按钮"]
    C --> D["修改内容"]
    D --> E["重新发送"]
    B -->|"管理上下文"| F["打开上下文面板"]
    F --> G["移除不需要的消息"]
    G --> H["AI 仅基于剩余上下文回复"]
```

---

### 场景 5: 审查者导入项目文件

**角色**: 代码审查者
**目标**: 将本地项目 ZIP 包导入到会话中供审查

```mermaid
flowchart LR
    A["点击导入按钮"] --> B["选择 ZIP 文件"]
    B --> C["上传到服务器"]
    C --> D["解压并关联到会话"]
    D --> E["文件树刷新"]
    E --> F["浏览导入的文件"]
```

**错误恢复**: ZIP 格式无效时提示错误；上传中断时可重新选择
