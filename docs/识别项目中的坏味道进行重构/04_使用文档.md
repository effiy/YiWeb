## 快速开始

### 概述

本次重构主要解决了 YiWeb 项目中的代码坏味道问题，包括全局状态污染、巨型文件、长参数列表等。重构采用渐进式策略，保持向后兼容。

### 重构范围

| 模块 | 说明 | 状态 |
|------|------|------|
| 全局注册机制 | 使用 Registry 替代 window 挂载 | ✅ 已设计 |
| 常量管理 | 提取魔法数字为常量 | ✅ 已设计 |
| 拖拽模块 | 从 aicr/index.js 拆分 | ✅ 已设计 |
| 上下文容器 | 简化依赖传递 | ✅ 已设计 |
| 监听器管理 | 统一事件监听管理 | ✅ 已设计 |

## 功能使用指南

### 1. 使用 Registry（注册中心）

#### 问题

之前代码中直接使用 `window[name]` 注册组件，造成全局污染。

#### 新用法

```javascript
import { registry } from '/cdn/utils/view/registry.js';

// 注册组件
registry.register('YiButton', component, {
    eventName: 'YiButtonLoaded'
});

// 获取组件
const YiButton = registry.get('YiButton');

// 分发事件
registry.dispatch('CustomEvent', { data: 'value' });
```

#### 向后兼容

仍然可以使用 `window.YiButton` 访问组件，旧代码无需修改。

---

### 2. 使用 Constants（常量模块）

#### 问题

代码中散落着大量魔法数字，难以维护。

#### 新用法

```javascript
import { TIMEOUTS, DEBOUNCE, CACHE } from '/cdn/utils/core/constants.js';

// 使用预定义的超时时间
await waitForComponents(components, TIMEOUTS.COMPONENT_LOAD);

// 使用防抖时间
setTimeout(callback, DEBOUNCE.SEARCH_INPUT);

// 使用缓存配置
const maxAge = CACHE.TEMPLATE_MAX_AGE;
```

#### 常量列表

| 常量组 | 常量名 | 值 | 说明 |
|--------|--------|-----|------|
| TIMEOUTS | COMPONENT_LOAD | 60000 | 组件加载超时 |
| TIMEOUTS | POLL_FAST | 50 | 快速轮询间隔 |
| TIMEOUTS | POLL_NORMAL | 100 | 普通轮询间隔 |
| TIMEOUTS | LOG_INTERVAL_SHORT | 2000 | 短日志间隔 |
| TIMEOUTS | LOG_INTERVAL_LONG | 5000 | 长日志间隔 |
| DEBOUNCE | SIDEBAR_RESIZE | 50 | 侧边栏防抖 |
| DEBOUNCE | SEARCH_INPUT | 300 | 搜索输入防抖 |
| CACHE | TEMPLATE_MAX_AGE | 604800000 | 模板缓存 7 天 |

### 3. 使用 Resizer（拖拽条工具）

#### 问题

之前拖拽条逻辑直接写在 aicr/index.js 中，代码过长。

#### 新用法

```javascript
import { createSidebarResizers, createResizer } from '/src/views/aicr/utils/resizer.js';

// 为 AICR 页面创建所有拖拽条
createSidebarResizers(store);

// 自定义单个拖拽条
const element = document.querySelector('.my-element');
createResizer(element, store, 'my-type', {
    minWidth: 100,
    maxWidth: 500,
    defaultWidth: 200,
    storageKey: 'my-element-width',
    saveWidth: (width) => { /* 保存逻辑 */ },
    position: 'right', // 或 'left'
    enforceLimits: true
});
```

#### 配置选项

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| minWidth | number | 240 | 最小宽度 |
| maxWidth | number | 400 | 最大宽度 |
| defaultWidth | number | 320 | 默认宽度 |
| storageKey | string | - | localStorage 键名 |
| saveWidth | function | - | 保存宽度回调 |
| position | string | 'right' | 拖拽条位置 |
| enforceLimits | boolean | false | 是否强制限制 |

### 4. 使用 ListenerManager（监听器管理器）

#### 问题

之前事件监听器代码重复，且容易忘记清理。

#### 新用法

```javascript
import { createListenerManager } from '/src/views/aicr/utils/listenerManager.js';

const listeners = createListenerManager();

// 添加监听器
listeners.add('keydown', (e) => {
    if (e.key === 'Escape') {
        // 处理 ESC
    }
});

listeners.add('highlightCodeLines', (e) => {
    // 处理高亮事件
});

// 挂载到全局方便清理
window.__aicrListenerManager = listeners;

// 清理所有监听器
window.__aicrListenerManager?.cleanup();
```

### 5. 使用 Context（应用上下文）

#### 问题

之前 useMethods 中参数传递冗长，每个函数都要传十几个参数。

#### 新用法

```javascript
import { createAppContext } from '/src/views/aicr/hooks/context.js';

// 创建上下文
const ctx = createAppContext(store);

// 传递给方法创建函数
const sessionChatContextMethods = createSessionChatContextMethods(ctx);
const folderTransferMethods = createFolderTransferMethods(ctx);
const authDialogMethods = createAuthDialogMethods(ctx);
// ... 等等

// 在方法内部使用
function createSessionChatContextMethods(ctx) {
    return {
        sendMessage: async (text) => {
            return ctx.safeExecuteAsync(async () => {
                const response = await ctx.postData(
                    ctx.getPromptUrl(),
                    { text }
                );
                return response;
            }, 'sendMessage');
        }
    };
}
```

#### 上下文对象属性

| 属性 | 说明 |
|------|------|
| `ctx.store` | 状态存储 |
| `ctx.safeExecute` | 安全执行函数 |
| `ctx.safeExecuteAsync` | 安全执行异步函数 |
| `ctx.getData` | GET 请求 |
| `ctx.postData` | POST 请求 |
| `ctx.SERVICE_MODULE` | 服务模块常量 |
| `ctx.buildServiceUrl` | 构建服务 URL |
| `ctx.getApiBaseUrl` | 获取 API 基础 URL |
| `ctx.getPromptUrl` | 获取提示 URL |
| `ctx.renderMarkdownHtml` | Markdown 渲染 |
| `ctx.renderStreamingHtml` | 流式渲染 |

## 迁移指南

### 从旧代码迁移

#### 1. 迁移 window 组件注册

```javascript
// 旧代码
window.YiButton = component;
window.dispatchEvent(new CustomEvent('YiButtonLoaded'));

// 新代码
import { registry } from '/cdn/utils/view/registry.js';
registry.register('YiButton', component, {
    eventName: 'YiButtonLoaded'
});
```

#### 2. 迁移魔法数字

```javascript
// 旧代码
await waitForComponents(components, 60000);

// 新代码
import { TIMEOUTS } from '/cdn/utils/core/constants.js';
await waitForComponents(components, TIMEOUTS.COMPONENT_LOAD);
```

#### 3. 迁移长参数列表

```javascript
// 旧代码
const sessionChatContextMethods = createSessionChatContextMethods({
    store,
    safeExecute,
    postData,
    SERVICE_MODULE,
    renderMarkdownHtml,
    renderStreamingHtml,
    getPromptUrl
});

// 新代码
import { createAppContext } from '/src/views/aicr/hooks/context.js';
const ctx = createAppContext(store);
const sessionChatContextMethods = createSessionChatContextMethods(ctx);
```

## 常见问题

### Q1: 重构后旧代码还能用吗？

**A**: 可以。所有重构都保持向后兼容，旧代码无需修改即可继续工作。

### Q2: 必须立即迁移到新 API 吗？

**A**: 不需要。可以渐进式迁移，新代码使用新 API，旧代码保持不变。

### Q3: 重构会影响性能吗？

**A**: 不会。重构主要是代码组织优化，不会引入额外的性能开销。

### Q4: 如何确认重构成功？

**A**: 检查以下几点：
1. 控制台无错误
2. 所有功能正常工作
3. 代码更清晰易读
4. 查找功能更快

### Q5: 遇到问题如何回滚？

**A**: 使用 Git 回滚：
```bash
# 查看提交历史
git log

# 回滚到特定提交
git reset --hard <commit-hash>
```

## 最佳实践

### 1. 新增代码使用新架构

```javascript
// ✅ 推荐
import { registry } from '/cdn/utils/view/registry.js';
import { TIMEOUTS } from '/cdn/utils/core/constants.js';

// ❌ 不推荐（旧方式）
window.MyComponent = ...;
setTimeout(..., 50);
```

### 2. 逐步迁移旧代码

不必一次性迁移所有代码，可以按模块逐步迁移。

### 3. 添加类型注释（JSDoc）

```javascript
/**
 * @param {import('./context').AppContext} ctx
 */
function createMyMethods(ctx) {
    // ctx 有自动补全
}
```

### 4. 保持文件小而专注

每个文件不超过 300 行，每个函数不超过 50 行。
