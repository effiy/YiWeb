## 功能概述

AICR 页面的头部区域目前由两行组成：上方是 SearchHeader（含首页按钮、搜索框、操作按钮），下方是 session-list-tags（含标签搜索、筛选按钮、标签列表）。两行布局占据了较多垂直空间，影响主内容区的可用面积。本次功能将这两行合并为一行水平布局，在桌面端让搜索框和标签筛选器并排显示，大幅减少头部占用高度，同时在平板和移动端回退为垂直堆叠保持可用性。

当前布局根因：`SearchHeader` 内部 `.header-row` 使用 `display: grid; grid-template-columns: auto 1fr auto`，其 `1fr` 列撑满可用宽度，导致 `search-header` 独占整行（来源：`cdn/components/business/SearchHeader/index.css:L6-19`）。

**核心价值**
- 🎯 减少头部垂直占用约 40-50%，释放更多空间给代码审查主内容区
- ⚡ 搜索与标签筛选可并行操作，无需上下切换焦点
- 📖 响应式设计确保各屏幕尺寸下布局合理降级

## 功能详情

> 本章写"要做什么/不做什么"，不写实现细节（实现细节放 `03_设计文档.md`）。

### 桌面端单行布局

- **功能说明**：在屏幕宽度 >1024px 时，SearchHeader（首页按钮+搜索框+操作按钮）与 session-list-tags（标签搜索+筛选按钮+标签列表）在同一水平行内并排显示
- **边界与异常**：SearchHeader 为 CDN 共享组件（`cdn/components/business/SearchHeader/`），不可修改其源码；通过 CSS 穿透选择器覆盖其内部布局
- **价值/动机**：减少头部垂直空间占用，让代码审查区域更大

---

### 平板/移动端响应式回退

- **功能说明**：在屏幕宽度 <=1024px 时，两区域回退为上下垂直堆叠布局，与当前行为一致
- **边界与异常**：移动端（<=768px）触摸目标尺寸需 >=44px，确保可点击性
- **价值/动机**：小屏幕下水平空间不足以并排两个组件，垂直堆叠更合理

---

### 标签拖拽排序适配

- **功能说明**：桌面端水平布局下，标签拖拽排序的方向判断从纵向（top/bottom）适配为横向（left/right）
- **边界与异常**：移动端回退为纵向布局后，拖拽判断需随布局方向切换
- **价值/动机**：拖拽排序是标签管理的核心交互，必须在新布局下保持正常

---

### 主内容区高度自适应

- **功能说明**：代码审查主内容区（`.aicr-main`）的高度应随 header 高度变化自动适配，不依赖硬编码的 `calc(100vh - Xpx)` 计算
- **边界与异常**：当前 `aicrPage/index.css` 和 `index.css` 中存在硬编码 header 高度值（64px/56px/48px），需改为弹性布局
- **价值/动机**：避免 header 高度变化后主内容区出现空白或溢出

## 验收标准

### P0 - 必须通过
- [ ] **桌面端单行显示**：在 >1024px 宽度下，SearchHeader 与 session-list-tags 在同一水平行显示，无换行
- [ ] **头部高度缩减**：桌面端 `.aicr-header` 的 offsetHeight 相比改动前减少 40%以上
- [ ] **主内容区无异常**：代码审查区域正确填满剩余高度，无空白区域或内容溢出
- [ ] **搜索功能正常**：搜索框输入、搜索、清除功能不受布局变更影响
- [ ] **标签筛选正常**：标签选择、反向过滤、无标签筛选、清除过滤功能不受影响

### P1 - 应该通过
- [ ] **拖拽排序适配**：桌面端水平布局下拖拽标签排序正常，插入指示方向正确
- [ ] **平板端回退**：768-1024px 宽度下两区域垂直堆叠，与改动前一致
- [ ] **移动端触摸友好**：<=768px 下按钮和标签最小触摸目标 >=44px

### P2 - 可以有
- [ ] **CSS 选择器特异性**：通过增加 `.aicr-header .session-list-tags` 前缀避免与 `fileTreeTags.css` 潜在冲突
- [ ] **SessionListTags 独立组件标注废弃**：在 `sessionListTags/index.js` 中添加 @deprecated 注释

## 使用场景示例

#### 📋 场景一：桌面端高效代码审查

> **背景**：开发者在 1440px 宽度的外接显示器上使用 AICR 进行代码审查
>
> **操作**：打开 AICR 页面，在搜索框输入关键词的同时，在右侧标签区点击标签进行筛选
>
> **结果**：头部区域紧凑地显示在一行内，搜索框和标签区并排，代码审查区域占据更多屏幕空间

---

#### 🎨 场景二：移动端浏览审查记录

> **背景**：开发者在手机上快速浏览代码审查记录
>
> **操作**：在 375px 宽度的手机上打开 AICR，看到搜索框和标签区上下排列
>
> **结果**：布局自动回退为垂直堆叠，触摸目标足够大，操作流畅
